## 1. 产品概述

### 背景与机会

Duolingo 正处于用户信任危机峰值：
- "Duolingo is lying to learners" — **4,791 upvotes**（r/duolingo 历史级高赞）
- "AI-First Disaster" — **3,391 upvotes**，用户大规模取消订阅
- "feels like malware" — **3,042 upvotes**
- "After 11.5 years, I'm giving up" — **1,648 upvotes**

最直接的竞品机会：Lingonaut（社区驱动替代品，alpha 阶段，**Android only**，无 iOS）获 **1,733 upvotes** 支持。

### 产品定位

**LinguaPath** = 语法优先的 iOS 语言学习应用，面向被 Duolingo "游戏化陷阱"驱逐的严肃学习者。

---

## 2. 目标用户画像

### 主要用户：被 Duolingo 放弃的严肃学习者

- **年龄**：22–40 岁
- **特征**：
  - 已有 Duolingo 300–1300 天 streak，但感到语言没有实质进步
  - 明确知道自己想要"真正学会"而非"保持 streak"
  - 有付费意愿（"$80/month on learning apps" 帖 1169 upvotes 验证）
  - iOS 用户（iPhone 主力人群，付费能力更强）
- **核心 JTBD**："我想在 12 个月内能用目标语言进行真实对话，而不是拿一个 streak 数字自我安慰"

### 次要用户：语言学习进阶者

- B1 以上已入门，寻找从 B1 推进到 B2/C1 的结构化路径
- 现有 app 几乎全部止步于 B1（Reddit 高频投诉）

### 差异化切入点

1. **语法笔记随课程内嵌**（每个知识点配 "为什么这样说" 的解释）
2. **CEFR A1→C2 完整路径**（业内唯一全覆盖 + 结构清晰）
3. **零暗黑设计**：无 Hearts/能量/强制付费墙，学习节奏自主
4. **iOS 原生体验**：SwiftUI，0.8s 启动，流畅体验（相比 RN 的 3.2s）
5. **Lingonaut 的 iOS 版**：主动定位为"iOS 上的 Lingonaut"，承接已有需求热度

### AI口语类竞品补充分析

> 核心发现：最大市场空白 = **「结构化路径 + 实际开口」**，无竞品同时实现。

| 竞品 | 核心定位 | 致命缺陷 |
|------|---------|---------|
| Speak | AI口语练习 | 无结构路径，用户不知道自己在哪个水平 |
| ELSA Speak | 发音/口音训练 | 功能孤立，无法进阶到真实对话 |
| Fluently | AI会话导师 | 缺配套语法+阅读模块 |
| BoldVoice | 美音口音训练 | 独立工具，无整合学习路径 |
| LingQ | 沉浸式阅读 | 学习曲线陡，普通用户难坚持 |
| **LinguaPath** | **路径+输出一体** | **填补以上所有缺口** |

**Reddit高赞痛点（AI口语App调研）**：
1. **学了用不出来**（2305↑）— 知道语法/词汇，真实场景开不了口
2. **口语焦虑** — 非能力问题，是「怕出错被评判」的心理障碍
3. **公共场合无法练习** — 地铁/办公室不敢录音，现有AI口语App全要求开麦
4. **B1后进阶断层** — 所有竞品止步于B1，无结构化进阶工具

## 6. 盈利模式与定价策略

### 三层变现漏斗（Wiki 验证模型）

```
免费用户（90%）
├── 基础课程 A1-B1 完全免费（无广告，无Hearts）
└── 变现：订阅引导（Paywall 放在 B2 入口）

订阅用户（8-9%）
├── $7.99/月 or $54.99/年
└── 解锁：B2-C2 全课程 + AI对话 + 发音评分 + 无限单词本

高端 IAP（1-2%）
├── $29.99 一次性：终身解锁（生命周期活跃用户的锁定策略）
└── $9.99/次：特定语言课程包（如商务日语、旅游法语）
```

### 定价逻辑

| 参数 | 数值 | 依据 |
|------|------|------|
| 月订阅 | **$7.99** | 比 Babbel ($12.95) 便宜 38%，比 Duolingo Super ($6.99) 贵但有语法 |
| 年订阅 | **$54.99** | = 月付 $4.58，折扣 43%，提升 LTV |
| 终身 IAP | **$79.99** | 等同 Duolingo 一年价格，长期用户锁定 |
| 课程包 | **$9.99** | 单语言轻量购买，降低付费门槛 |

### 收入预测（MVP后12个月）

```
下载量目标: 50,000（iOS，北美 + 欧洲市场）

## 7. 技术选型

| 决策 | 选择 | 理由 |
|------|------|------|
| **平台** | iOS 优先（SwiftUI） | 用户付费能力 2-3x Android；3个月 MVP 可达 |
| **框架** | SwiftUI | 500行 vs RN 5000行；启动 0.8s vs 3.2s |
| **后端** | Supabase | 开源 Firebase 替代，免费额度足够 MVP |
| **AI** | OpenAI GPT-4o mini API | 对话练习 + 语法解释生成，成本 ~$0.01/次 |
| **语音** | Apple Speech Framework | 原生 iOS，零额外成本，隐私友好 |
| **分析** | PostHog（自托管） | 免费，行为追踪 + A/B 测试 |

### MVP 技术架构

```
iOS SwiftUI App
├── 离线优先（课程内容本地 SQLite）
├── 云同步（Supabase Realtime）
└── AI模块（OpenAI API，仅联网功能）
```

## 11. 功能模块详细设计

> 完整设计文档：`/Users/zhangyan/Projects/app_project/docs/plans/2026-04-12-linguapath-module-design.md`

### 核心设计决策汇总

| 模块 | 设计方案 | 理由 |
|------|---------|------|
| 语法解释 | 答题后内嵌卡片（三层深度） | 不打断节奏，按需展开 |
| 激励机制 | 能力成长可视化（CEFR进度条） | 真实能力感 > streak焦虑 |
| AI对话纠错 | 对话后汇总报告 | 保持沉浸感，错误有上下文 |
| Onboarding | 10题自适应分级测试 | 避免重学已知内容 |

### 一节课的形态（10-15分钟）
```
导入（90s）→ 练习（8-10min）→ 语法复盘（2min）→ 课后总结（30s）
```
5种练习模式：听→选 / 译→填 / 排列组词 / 听写 / 翻译输出

### CEFR 升级机制
- 每个语法点独立掌握度分数（0-100），遗忘曲线自动衰减
- 全级别语法点 ≥70 → 解锁升级测试（20题，80%通过）
- 升级奖励：数字证书（可分享）= 主要社交传播钩子

### AI 对话场景分级
A1 自我介绍 → A2 购物/问路 → B1 面试/租房 → B2 辩论 → C1+ 自由话题

### Onboarding（4分钟，无强制注册）
选语言 → 选目标场景 → 10题分级测试 → 推荐起始课程 → 设定每日目标 → 直接进入第一课

## 12. 持续学习框架

> 完整设计文档：`/Users/zhangyan/Projects/app_project/docs/plans/2026-04-12-linguapath-learning-framework-design.md`
> 设计依据：Reddit学习效率与动力调研（2026-04-12）

### 核心设计决策

| 问题 | 方案 |
|------|------|
| Why锚定 | Onboarding选目标场景，所有内容围绕该场景定制 |

### 场景解锁进度（示例：旅行场景）
```
✅ 机场问路  ✅ 咖啡厅点餐  ✅ 预订酒店

### 积分系统
- 完成周挑战录音：150分 / 写作：100分
- 获赞：5分/赞（上限50）；给他人留言：10分
- 兑换：高级场景包（500分）/ AI角色皮肤（300分）/ 订阅折扣（1000分）

### 5分钟模式内容
复习卡（30s）→ 100词短文（90s）→ 3题填空（90s）→「今天也做到了 ✓」

## 14. Speaking Ladder（口语焦虑破冰系统）

### 设计原则
Speaking Ladder 是 AI自由对话的**前置解锁路径**，不是独立功能。
用户必须通过台阶①②③，才能解锁场景的AI自由对话奖励。

### 四台阶设计

```
台阶① 文字预演 → 朗读（有稿有评分）
  用户先打字写出要说的内容
  确认内容后按住录音朗读
  Apple Speech评分，通过即完成
  目标：分离"知道说什么"和"能说出来"两个障碍

台阶② 提示词 → 半开放回答
  屏幕显示关键词提示（非完整句子）
  用户自由组织，录音输出
  目标：从复现句子过渡到自主生成

台阶③ 场景填空对话
  AI给出对话上半句，用户接下半句
  例："I heard you went to Tokyo. How was ___?"
  目标：真实对话节奏，有明确语境支撑

台阶④ AI自由对话（= 场景解锁奖励）
  无稿、无提示的真实交流
  对话后汇总报告（错误+语法卡+可加入专项练习）
```

### 与场景解锁的绑定
```
台阶① 完成3次 → 解锁场景基础AI对话权限
台阶② 完成3次 → 解锁场景进阶AI对话
台阶③ 完成3次 → 场景完全解锁（触发解锁动画）
台阶④ = 解锁后的奖励体验入口
```

**进阶规则**：连续3次满分 → 解锁下一台阶。可随时退回上一台阶，无惩罚。

## 15. 静音输出训练（Silent Output Mode）

### 触发方式
5分钟模式顶部新增手动开关：`[💪 正常模式]  [🔇 静音模式]`
用户上次选择作为下次默认，记忆偏好。

### 静音模式结构
```
正常模式：复习卡(30s) → 短文阅读(90s) → 填空3题(90s) → 完成✓
静音模式：复习卡(30s) → 短文阅读(90s) → 限时打字输出(90s) → 完成✓
```

### 打字输出环节设计
- 给出场景提示词（如："朋友问你推荐一部最近看的电影"）
- 限时60秒，打字输出，越多越好
- 完成后显示：
  - 词数统计（目标：B1=30词/分钟）

## 17. 分级学习方法（Level-Adaptive Learning）

### 核心逻辑
- **B1前**：输入主导（70%输入 / 30%输出）

- 题型不淘汰，升级后旧题型变为热身/复习用途
- 4段式课程结构不变，"练习"段内部配比动态调整

| CEFR | 练习段构成 | 输出比例 | 本级新技能 |

| C1 | 长文分析30% + AI自由辩论40% + 写作润色30% | 85% | 语体切换+长文分析 |

**意图**：让目标语言不是英语的用户提前建立期待，增加非英语市场的订阅意愿，同时为扩展阶段提供需求密度数据参考。

### 内容架构原则：框架共享，内容独立

| 层级 | 共享/独立 | 说明 |
|------|---------|------|
| 产品框架层 | ✅ 共享 | 场景地图、解锁条件、积分规则、CEFR分级逻辑 |
| 语法体系 | ❌ 独立 | 日语助词体系 vs 西语动词变位，结构差异过大 |
| 故事库 | ❌ 独立 | 每门语言独立编写，与该语言语境高度绑定 |
| 练习内容 | ❌ 独立 | 题库、例句、词汇表均语言专属 |
| AI对话Prompt | 部分共享 | 场景设定共享，语言规则独立 |

## 27. 练习系统设计（掌握度驱动渐进路径）

> 本节记录 2026-04-13 brainstorming 确定的练习系统架构，作为 P0 实现依据。

### 27.1 设计原则

- **听说读写渐进**：新语法点从"听"开始，随掌握度提升逐步解锁"写"和"说"
- **CEFR × 场景双维度选题**：题目由用户当前 CEFR 级别 + 选择的学习场景共同决定
- **掌握度自适应**：同一语法点，不同用户可能处于不同阶段，防止低价值重复
- **无心心制**：答错直接重练，不扣命值

### 27.2 掌握度阶段（masteryScore 驱动）

| 阶段 | masteryScore 区间 | 解锁题型 | 技能方向 |
|------|-----------------|---------|---------|
| 入门 | 0 – 25 | listenPick | 听 |
| 识别 | 26 – 50 | listenPick + translateFill | 听 + 读 |
| 输出 | 51 – 75 | translateFill + wordShuffle | 读 + 写 |
| 精通 | 76 – 100 | wordShuffle + dictation + openTranslation | 写 + 说 |

### 27.3 每节课选题流程（SessionBuilder）

```
用户 CEFR 级别 + 学习场景
        ↓
从 CoreData 取该 level+scene 下掌握度最低的 3 个语法点
        ↓
每个语法点按其 masteryScore 阶段生成 3–4 道题
        ↓
共 ~12 题，按语法点 block 顺序排列
        ↓
答完一个 block → 更新 masteryScore → 触发 GrammarCard
```

### 27.4 题库规模（ExerciseBank）

- **覆盖范围**：A1 + A2 × Travel / Work / Culture / Social = ~40 个语法点
- **每语法点**：每阶段 2 道题变体 × 4 阶段 = ~8 道题
- **总题量**：~320 道题
- **存储方式**：静态 Swift 数据，三级索引 `[cefrLevel][scene][grammarPointKey]`

### 27.5 CoreData 变更

- `GrammarPoint` 实体新增 `scene: String` 字段（对应 LearningScene.rawValue）

### 27.6 新增组件

| 组件 | 职责 |
|------|------|
| `ExerciseBank.swift` | 静态题库，三级索引结构 |
| `SessionBuilder.swift` | 从 CoreData 取语法点 + ExerciseBank 组装课程 |
| `ListenPickView.swift` | 听后选择正确句子 |
| `TranslateFillView.swift` | 看中文提示，输入英文翻译 |
| `WordShuffleView.swift` | 拖拽排列单词成正确句子 |
| `DictationView.swift` | 听音频，输入听到的内容 |
| `OpenTranslationView.swift` | 自由翻译，AI 或关键词评分 |

### 27.7 与现有代码关系

- `ExerciseFactory` → 替换为 `SessionBuilder.buildSession(level:scene:persistenceController:)`
- `LessonViewModel.completePhase(.introduction)` 改为调用 `SessionBuilder`
- `LessonExerciseView` 中的占位符替换为 5 种具体 Exercise View
- `ExerciseViewModel` 保持不变（答题逻辑复用）
