# 用户体验与 Core Web Vitals

> 谷歌已将用户行为信号作为重要排名因素。测验/问答网站天然具有高互动潜力，正确设计可将 UX 优势转化为 SEO 红利。

![[用户体验与Core Web Vitals.png|600]]

## 1. 用户行为信号与排名的关系

谷歌通过 Chrome 真实用户数据追踪：

| 信号 | 含义 | 优化方向 |
|------|------|---------|
| **CTR（点击率）** | 搜索结果中的点击 | 优化 Title + Description |
| **停留时长（Dwell Time）** | 用户在页面的时间 | 嵌入视频、测验、互动工具 |
| **跳出（Pogo-sticking）** | 点击后迅速返回 | 首屏立即传递价值 |
| **页面参与度** | 向下滚动、点击互动 | 结构清晰、内容扫描友好 |

> [!tip] 问答网站优势
> 测验天然产生高停留时长（用户需要完成所有题目），这是 UX 信号的天然加分项。

## 2. CTR（点击率）优化策略

### Title 标题优化
- 包含**精确匹配的核心关键词**（谷歌有 title match score 信号）
- 长度：55-60 字符（避免截断）
- 加入情感词或数字：「5分钟」「免费」「2025年最新」
- ❌ 不要在标题放电话号码（阻止用户点击进入网站）

### Meta Description 优化
- 包含主要关键词（会被粗体高亮）
- 160 字符以内
- 明确说明用户能获得什么（Call-to-Value）
- 测验网站模板：`「完成这个 [主题] 测验，发现你是哪种 [类型]。仅需3分钟，免费测试！」`

### 富摘要（Rich Snippets）CTR 提升
- FAQ Schema → 展开式问答摘要 → 更大占地面积
- Quiz Schema → 星级/互动标记
- BreadcrumbList → 清晰的导航路径

## 3. 停留时长优化

### 对问答网站的专属策略

**测验流程设计：**
- 进度条显示（「第 3/10 题」）→ 鼓励完成
- 即时反馈（选择后显示是否正确）→ 提升参与感
- 引人入胜的结果页（个性化分析 + 分享按钮）

**结果页延长停留：**
- 显示「你的得分高于 X% 的参与者」
- 推荐相关测验（「你可能也感兴趣：...」）
- 鼓励分享后返回（「分享给朋友挑战你！」）

### 多媒体内容嵌入
- 在测验详情页嵌入相关解说视频
- 微型计算器（如「你知道多少？」评分预测）
- 图表/信息图解释测验背景

## 4. 网站导航与用户路径

### 首页设计原则
- 首屏：清晰的分类入口 + 热门测验推荐
- 搜索框突出（用户主动探索）
- 热门/新上线测验模块

### 分类页优化
- 分类描述包含目标关键词
- 测验卡片：标题 + 简短描述 + 已参与人数
- 筛选/排序功能（按热度/难度/最新）

### 测验详情页结构
```
H1: [测验名称]（含核心关键词）
[40-60字简介：说明测验目的和预期收获]
[CTA按钮：开始测验]
[测验概述：题目数量、所需时间、主题说明]
[测验内嵌组件]
[结果说明区]
[FAQ Schema标注的常见问题]
[相关测验推荐]
```

## 5. 移动端 UX 优化

测验网站流量中移动端通常占 60-70%：

- 单列布局，避免横向滚动
- 选项按钮高度 ≥ 44px（手指点击友好）
- 进度条固定在顶部（随滚动可见）
- 分享按钮在结果页突出且易点击

## 6. 页面速度 × UX 的乘数效应

速度提升 1 秒 → 转化率提升约 7%（Google 数据）

对测验网站：
- 题目加载慢 → 用户放弃 → 停留时长骤降
- 结果页加载慢 → 分享率下降 → 自然外链损失
