# 埋点行为特征体系

基于用户行为埋点数据构建的特征体系，覆盖二次风控（二期697特征）和运营模型（三期T+1离线特征81个）两大场景。

---

## 核心概念

### 事件表

- **主埋点事件表**：`ec_dwd.dwd_ec_log_di_sa_app_events`
  - 关键字段：`event_code`, `content`(JSON), `created_time`, `loan_account_id`, `dt`
- **风控人群表**：`dm_id.dm_id_risk_df_underwrite`

### 时间窗口体系

采用**累积包含**方式定义时间窗口，较长窗口包含所有较短窗口：

| 时间维度 | 包含范围 | 示例 |
|---------|---------|------|
| 1h | ('1h') | `switch_out_cnt_1h` |
| 3h | ('1h','3h') | `switch_out_cnt_3h` |
| 6h | ('1h','3h','6h') | `switch_out_cnt_6h` |
| 12h | ('1h','3h','6h','12h') | `switch_out_cnt_12h` |
| 1d | ('1h','3h','6h','12h','1d') | `switch_out_cnt_1d` |
| 3d | ('1h','3h','6h','12h','1d','3d') | `switch_out_cnt_3d` |
| 7d | ('1h','3h','6h','12h','1d','3d','7d') | `switch_out_cnt_7d` |

运营模型额外窗口：14d / 28d / 56d / 112d / all（历史全量）。

### 关键事件代码

| 事件代码 | 描述 | 风险含义 |
|---------|------|---------|
| `general_switch_out_result` | 应用切出 | 远程控制检测 |
| `general_switch_in_result` | 应用切入 | 远程控制检测 |
| `general_b396_pageview` | 截屏 | 信息窃取检测 |
| `login_login_result` | 登录 | 账户安全 |
| `AppStart` / `AppEnd` | 应用启动/退出 | 使用模式分析 |

### 特征命名规范

```
{特征类型}_{事件/字段}_{统计维度}_{时间窗口}

例如:
- switch_out_cnt_7d           -- 切出次数，7天
- ip_switch_cnt_3h            -- IP切换次数，3小时
- app_start_night_ratio_7d    -- APP启动夜间占比，7天
- interval_avg_switch_out_7d  -- 切出平均间隔，7天
```

**常用后缀**：`_cnt`(计数) / `_ratio`(占比) / `_max`/`_min`/`_avg` / `_freq2`/`_freq3`(频繁操作) / `_1h~_7d`(时间窗口)

## 15种特征逻辑模式

### 模式1: 时间窗口统计

对同一事件在不同时间窗口进行计数。统计事件包括：切出/切入/截屏/登录/APP启动/APP退出，共6类 x 7窗口。

**算子配置**：`event_code + time_window + filter_condition`

### 模式2: 页面分布统计

对特定页面/URL进行计数，识别用户在不同页面的行为分布：
- 切出页面分布（Top 10）：首页/账单/贷款输入/启动页等
- 切入页面分布（Top 10）
- 截屏页面分布（Top 5）
- 高风险截屏页面（征信报告/贷款协议/电子签章/提现/还款等）

### 模式3: 事件属性统计

基于事件属性分析操作方式和结果：
- 登录方式分布（验证码/密码/生物识别）
- 登录验证方式（WhatsApp/SMS）
- 登录结果（成功/失败/密码错误/验证码错误/OTP错误）
- 修改密码/换绑手机/银行卡操作/认证结果统计

### 模式4: 时间间隔统计

计算同一事件连续发生的时间间隔（使用 `LEAD()` 窗口函数）：
- 切出/切入间隔的 avg/max/min
- 各页面的间隔统计

### 模式5: 异常检测（短时高频行为）

检测 10s/1min/5min bucket内的异常高频行为：
- 同一事件在同一bucket内出现 >=2次 -> freq2
- 同一事件在同一bucket内出现 >=3次 -> freq3
- 覆盖切出/切入/APP启动/截屏/登录 5类事件

### 模式6: IP/地理/设备/网络统计

使用 `LAG()` 窗口函数检测切换行为：
- IP切换频率（7个时间维度 + 切换率）
- 城市切换频率
- 网络切换频率（WiFi/4G/5G）
- 设备切换频率（fin_device_id）
- 不同值数量（distinct IP/城市/网络/设备）

### 模式7: 时间分布统计

按时段和星期分布统计：
- 夜间(0-6) / 日间(6-18) / 晚间(18-24)
- 工作日(周一至五) / 周末
- APP启动/登录/切出 的时段分布和占比
- 会话时长统计（快速退出/平均/最大/最小时长）

### 模式8~15: 扩展模式

| 模式 | 说明 | 特征数 |
|------|------|--------|
| 比例计算 | 登录成功占比、验证方式占比等 | ~45 |
| 反欺诈模式 | 风险序列、账户变更 | ~43 |
| 每日频率 | 日级敏感操作频率 | ~7 |
| 跨事件关联 | 改密后登录、登录后截屏等 | ~7 |
| 设备指纹 | 同device_id跨用户共享 | ~7 |
| 模拟器/Root检测 | 低风险设备、NULL网络 | ~8 |
| 短时操作 | 1分钟/5分钟内操作密度 | ~10 |
| 风险行为序列 | 换绑+改密组合检测 | ~9 |

## 运营埋点T+1特征

### 三期特征体系总览

基于263个稳定埋点事件，面向运营Uplift模型设计的81个T+1离线特征：

```
运营模型特征体系
|-- A 活跃与触达意愿特征（13个） -> 判断"值不值得今天触达"
|-- B 借款意向信号特征（15个）  -> 判断"触达能不能转化"
|-- C 认证进度特征（16个）      -> 针对首贷未完件人群
|-- D 贷后还款行为特征（13个）  -> 针对复贷人群
|-- E 营销响应偏好特征（10个）  -> 选择最优消息类型
|-- F 产品与价格敏感特征（6个） -> 选择最优消息内容
|-- G 风险信号特征（8个）       -> 识别Do-Not-Disturb人群
```

### A类：活跃与触达意愿（全部人群）

| 特征名 | 构建逻辑 | 窗口 |
|-------|---------|------|
| `wa_msg_auth_consent` | 是否授权WA消息推送 | 历史 |
| `homepage_pv_7d/30d` | 首页访问次数 | 7d/30d |
| `active_days_7d/30d` | 活跃天数 | 7d/30d |
| `recency_ratio_7_30` | 近7天/30天活跃天数比 | 衍生 |
| `popup_accept_rate_30d` | 弹窗响应率 | 30d |
| `homepage_product_card_click_7d` | 首页产品大卡点击 | 7d |

### B类：借款意向信号（首贷已完件+复贷）

| 特征名 | 构建逻辑 | 窗口 |
|-------|---------|------|
| `loan_orderpage_pv_7d/30d` | 下单页展示次数 | 7d/30d |
| `loan_submit_btn_click_7d` | 点击下单按钮 | 7d |
| `loan_amount_adjust_cnt_7d` | 额度滑杆调整 | 7d |
| `loan_repay_plan_view_cnt_30d` | 查看还款计划 | 30d |
| `loan_fee_detail_view_cnt_30d` | 查看费用详情 | 30d |
| `loan_intent_score_7d` | 借款意向综合得分 | 7d |

**意向综合得分公式**：
```
loan_intent_score_7d =
  orderpage_pv * 1 +
  submit_click * 3 +
  repay_plan_view * 2 +
  amount_adjust * 1.5 +
  coupon_click * 2
```

### C类：认证进度（首贷未完件专用）

核心特征：`auth_step_completion_score`（0~6分），按到达工作信息页/KTP准备页/KTP确认/活体识别/联系信息/绑卡各+1分。

### D类：贷后还款行为（复贷专用）

核心特征：`post_proactive_repay_score`（主动还款意愿得分）= bill_pv*1 + repay_click*3 + bubble_click*2

### G类：风险/负向信号

| 特征名 | 风险含义 |
|-------|---------|
| `loan_market_visit_cnt_30d` | 多头借贷倾向 |
| `rejected_then_loanmarket_flag` | 被拒后访问贷超=资金饥渴 |
| `repay_fail_recent_flag` | 近14天还款失败 |
| `wa_consent_reject_flag` | 明确拒绝WA授权 |
| `screen_record_detect_flag` | 屏幕录制=欺诈信号 |

## Kite算子配置

### 端到端流程

```
Hive SQL (离线特征开发)
    -> hive2mysql (字段映射 + 语法转换)
MySQL SQL (基于 sensor_risk_events_enriched_7d)
    -> sql2kite (意图理解 -> JSON生成)
Kite JSON Config (导入平台即可上线)
```

### 当前线上算子（13个，1598特征）

| 算子 | 特征数 | 说明 |
|------|--------|------|
| SENSOR_TIME_WINDOW_STATS | 67 | 6事件x7窗口+加速度+交叉比率 |
| SENSOR_PAGE_DISTRIBUTION | 114 | 切出/切入/截屏xTop+高风险页面 |
| SENSOR_EVENT_ATTR_STATS | 183 | 登录/密码/银行卡/认证x7窗口+比率 |
| SENSOR_INTERVAL_STATS | 66 | 事件间隔avg/max/minx页面x1d/3d/7d |
| SENSOR_ABNORMAL_FREQ | 840 | 14事件x4桶x2阈值x7窗口(shift邻桶) |
| SENSOR_IP_GEO_STATS | 40 | IP/城市/网络/设备切换x7窗口 |
| SENSOR_RATIO_STATS | 77 | 时段占比+快退+会话时长 |
| SENSOR_HIGHRISK_PAGE_TW | 112 | 高风险页面x7窗口 |
| SENSOR_SWITCH_BACK_TW | 28 | 切回检测x4页面x7窗口 |
| SENSOR_ACTIVE_DAYS | 43 | 活跃天数x5事件x7窗口 |
| SENSOR_SESSION_FEATURES | 9 | 会话数/时长/短长占比 |
| SENSOR_PAGE_DIVERSITY | 9 | 页面去重/分散度 |
| SENSOR_DEVICE_ENV | 10 | 设备/网络/平台分布 |

### 关键配置规则

- 严格序列化：所有可选字段必须显式声明 `null`
- uuid 必填
- 时间窗口用 `second_diff(trace_time_created, TIME_CREATED)` 计算距今秒数
- 异常频率检测用shift邻桶检测，不用GROUP_BY动态桶ID
- Kite不支持跨算子引用，用 AGGREGATE EXTEND 在同一算子内实现
