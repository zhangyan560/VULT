## 一、UBT变量挖掘

### 1.1 简介

**UBT（User Behavior Tracking）**：追踪用户在APP内的行为轨迹，挖掘与风险相关的行为特征。

挖掘维度：
- **基础页面UBT**：访问次数、停留时间、点击次数
- **重点页面UBT**：注册页面、活体页面、发标页面、还款页面等

使用方式：
1. 作为有显著贡献度的变量入模（如 y6 rank 9、rank 11 均为UBT变量）
2. 训练专项UBT模型用于策略发标拒绝

### 1.2 发标页面额度修改类变量（印尼模型文档）

以下为发标页面用户选择金额的行为变量体系：

#### 基本信息类

| 变量名 | 含义 |
|--------|------|
| `ubt_credit_lmt` | 最高额度 |
| `ubt_selected_amount` | 选择次数最多的额度 |
| `ubt_selected_amount_div_credit_ratio` | 选择次数最多的额度占总额度的比例 |
| `ubt_selected_amount_occurance` | 选择次数最多的额度的选取次数 |
| `ubt_max_tried_amount` | 用户选取额度时选取的最大值 |
| `ubt_min_tried_amount` | 用户选取额度时选取的最小值 |
| `ubt_avg_tried_amount` | 用户选取额度时选取的平均值 |
| `ubt_max_tried_amount_ratio` | 用户选取额度时选取的最大值占额度的比值 |
| `ubt_min_tried_amount_ratio` | 用户选取额度时选取的最小值占额度的比值 |
| `ubt_avg_tried_amount_ratio` | 用户选取额度时选取的平均值占额度的比值 |
| `ubt_max_tried_amount_occurance` | 最大值的选取次数 |
| `ubt_min_tried_amount_occurance` | 最小值的选取次数 |
| `ubt_avg_tried_amount_occurance` | 平均值的选取次数 |
| `ubt_sum_tried_amount_occurance` | 用户选取额度时的总选取次数 |

#### 修改行为类

| 变量名 | 含义 |
|--------|------|
| `ubt_amount_selection_behaviour` | 用户在发标调整额度时更多是降额还是升额 |
| `ubt_increase_amount_occurance` | 升额的次数 |
| `ubt_decrease_amount_occurance` | 降额的次数 |
| `ubt_inc_dec_amount_diff` | 升额降额次数之差 |
| `ubt_increase_amount_ratio` | 升额次数的占比 |
| `ubt_decrease_amount_ratio` | 降额次数的占比 |
| `ubt_max_max_amount_diff` | 最大额度调整跨度 |
| `ubt_min_min_amount_diff` | 最小额度调整跨度 |
| `ubt_avg_avg_amount_diff` | 平均额度调整跨度 |

> [!note] 业务逻辑
> 在发标页面，用户通过滑动条或输入选择借款金额。降额行为（用户主动选择比授信额度更小的金额）往往意味着用户资金压力较小、风险较低；而反复修改或者选择最大额度的行为可能与高风险相关。

### 1.3 UBT在发标模型中的应用

在[[优选贷发标模型y6]]中，UBT变量排名：
- `loan_term_select_before_10080m_clk_count`（发标页面期限点击次数）：Gain排名 **第9**
- `loan_amount_before_10080m_clk_count`（发标页面金额点击次数）：Gain排名 **第11**

在[[优选贷发标模型y7]]中，继续保留上述变量，并新增了23年衍生的页面元素是否点击类变量（8个）。

## 二、UBT One-hot衍生 & UBT专项模型

**源文件**：`202310_变量衍生/ubtonehot衍生&模型.xlsx`

### 背景与结论

**背景**：
1. 老客运营风险模型中UBT变量有增益，尝试将相同方法用于新客
2. 需要挖掘戳额阶段可用的UBT变量

**结论**：
1. 本次UBT模型剔除了期限、金额相关的特征，考虑了戳额时刻与发标时刻的差异和随时间的稳定性，线上表现更稳定
2. 相较22年UBT变量，提升 **0.3个点** AUC
3. 单分来看：UBT戳额/发标时刻模型短期AUC较高（0.55~0.6），长期target效果一般
4. UBT发标时刻模型比UBT戳额时刻模型 AUC高 **1个点**
5. 新版期限在各bin都比较稳定（旧版因含期限相关衍生导致期限倒序）；金额排序新版比旧版更优

### 两种UBT模型

| 模型 | 标识 | 打分时刻 | 特点 |
|------|------|---------|------|
| 戳额时刻UBT模型 | ubtchuoe23 | 用户戳额时 | 短期AUC 0.55~0.6，覆盖更早 |
| 发标时刻UBT模型 | ubtfb23 | 用户发标时 | AUC比戳额版高约1个点，信息更丰富 |

## 三、手机号前7位地址风险分（城市风险评级）

**源文件**：`202310_变量衍生/手机号前7位地址风险分.xlsx`

### 方法论

利用手机号前7位可推断用户所属城市，基于历史成交数据计算每个城市的风险倍数，将城市分为6个风险等级。

### 样本

| 数据集 | 样本量 | 样本描述 | 时间范围 |
|--------|--------|---------|---------|
| 训练集 | 4,010,611 | 优选贷+极速贷成交样本 | 2021-01-01 ~ 2022-12-31 |

### 结论

1. 在训练集根据城市风险得到城市到风险倍数的映射，将城市根据风险分为 **6类**，训练集与测试集AUC差距在0.5个点以内
2. 城市风险评级在优选贷上的 AUC 可达 **0.54**，在极速贷上可达 **0.52**
3. 在优选贷上，城市风险评级头尾分别能找到 **0.7倍** 和 **1.35倍** 风险的人群
4. 在极速贷上，城市风险评级头尾分别能找到 **0.8倍** 和 **1.1倍** 风险的人群
5. 在优选贷上，与主模型分融合有 **0.2个点** 的AUC增益

### 变量名

`number_prefix_3`（手机号前3位/前7位城市映射风险分）— 在[[优选贷发标模型y7]]特征重要性中排名 **第7**

## 四、同一机型戳额次数

**源文件**：`202310_变量衍生/同一机型戳额次数.xlsx`

### 特征体系

| 变量名 | 含义 | 缺失率 |
|--------|------|--------|
| `same_device_chuoe_cnt` | 同一机型戳额次数 | 99.46% |
| `max_chuoe_diff` | 最近一次戳额时间与历史戳额时间的差的最大值 | 99.46% |
| `min_chuoe_diff` | 最近一次戳额时间与历史戳额时间的差的最小值 | 99.46% |
| `avg_credit_amount_old` | 历史戳额平均额度 | 99.46% |

> [!note] 缺失率说明
> 该特征缺失率极高（~99.5%），说明大多数用户只有一次戳额记录或无历史同机型戳额数据。该特征主要对少数多次戳额用户有效，可用于识别设备维度的欺诈行为（如同一手机型号被多人使用戳额）。
