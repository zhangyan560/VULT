# 信也工作概览

## 工作背景

**公司**：信也科技（XinYe Technology）

**主要产品**：
- **优选贷**：主力消费信贷产品，面向优质新客，额度较高（最高可达十万级别）
- **极速贷**：面向相对次优客群，流程更简化；优选贷拒绝人群会导流至极速贷

**整体流程**：用户戳额（申请授信）→ 数据源调用 → 分流模型（优选/极速分流）→ 主模型打分 → 评级生成 → 发标（授信决策/额度确定）

**评级体系**：
- 主模型（20个bin）× 规则分（5个bin）= 二维评级（A/B/C/D级别）
- 二维评级 × 额度分（7个bin）= 三维评级（最终用于发标策略）
- 评级矩阵为 20×5×7 结构

**主要数据源**：腾讯、百度、字节、百融、同盾、天创、银联智策、度小满等三方联合建模分，以及自有征信、多头、UBT行为数据

## 主要工作内容

### 1. 发标模型系列（核心工作）

发标模型作用于发标环节，主要功能为**拒绝**和**拆标（降额）**，是风控决策的最后一道关卡。

| 版本 | 时间 | 状态 |
|------|------|------|
| y4 | - | 已下线（仅用于线下估算拆标系数） |
| y5 | ~2023-01前后上线 | 已被y6替代 |
| y6 | 2023-08建模，2023年上线 | 已被y7替代 |
| y7 | 2024-05建模 | 当前线上版本 |
| y8 | - | 规划中/评估中 |

详见：[[发标模型体系]] | [[优选贷发标模型y6]] | [[优选贷发标模型y7]]

### 2. 变量衍生

- UBT行为变量挖掘（发标页面用户操作行为）
- One-hot衍生 & UBT模型（戳额/发标时刻）
- 手机号前7位地址风险分（城市风险评级）
- 同一机型戳额次数特征
- 交互式特征分析

详见：[[UBT用户行为特征]]

### 3. 其他模型

- 分流模型（一体化/非一体化，将用户分流至优选贷/极速贷流程）
- 主模型评估与迭代（优选贷主模型 v10/v11 等，迭代周期约6个月）
- 多模型（提额策略模型）
- 费率确认率敏感模型
- 极速贷额度策略模型

## 关键业务概念

| 术语 | 含义 |
|------|------|
| 戳额 | 用户申请授信，触发模型打分和数据源调用 |
| 发标 | 用户确认借款，发起标的（授信决策环节） |
| 拆标 | 将用户申请金额拆分为更小金额，降低单笔风险 |
| cap | 新旧模型bin之间的约束，防止新模型向上迁移过大（cap_bin = max(old_bin-4, new_bin)） |
| target_ever30_3m | 建模目标变量：3个月内是否出现30天以上逾期 |
| OOT | 跨时间验证集（Out-of-Time） |
| KS/AUC | 模型区分度指标 |
| sloping | 各分数段的风险梯度排序能力 |
