# 样本处理方法

风控建模中常用的样本处理技术，解决样本偏差、类别不平衡等问题。

---

## 拒绝推断

### 背景

信贷风控模型只能在"通过审核并放款"的用户上观察到违约标签，被拒绝的用户没有表现数据。这导致训练样本存在**选择偏差**（Sample Selection Bias）——模型只看到了"相对好"的人群，无法学习被拒绝人群中的风险模式。

### 方法：Deep Generative Models for Reject Inference

基于论文 *Deep Generative Models for Reject Inference in Credit Scoring*（Mancisidor et al., 2020, Knowledge-Based Systems）。

核心思路：使用深度生成模型（VAE变体）在潜空间中建模已标注样本（通过审核的用户）和未标注样本（被拒绝的用户），推断被拒绝用户的违约概率。

**两种模型架构**：
- **Model 1**：半监督VAE，联合建模标签和特征
- **Model 2**：在Model 1基础上加入GMM先验，潜空间呈混合高斯分布

### 使用方式

```bash
# 数据集：Lending Club
# supervised_lc_paper.pk.gz + unsupervised_lc_paper.pk.gz

# Model 1 训练（GPU）
THEANO_FLAGS=device=cuda0,floatX=float32 python -u training_model1.py \
    --outfile model1 --epochs 401 --n_cv 1 --beta 1.1 \
    --dset paper --n_sup 1552 --n_unsup 30997

# Model 2 训练（GPU）
THEANO_FLAGS=device=cuda0,floatX=float32 python -u training_model2.py \
    --outfile model2 --epochs 501 --n_cv 1 --beta 1.1 \
    --dset paper --n_sup 1552 --n_unsup 30997
```

关键超参数：
- `--n_sup`：少数类（y=1）观测数
- `--n_unsup`：无监督观测数
- `--zdim`：潜变量维度
- `--beta`：KL散度权重

**注意事项**：
- 训练不稳定，可能出现AUC骤降，需多次运行取最优
- 约500个epoch后潜空间才会展现混合分布结构
- GPU训练的权重只能在GPU加载，反之亦然

### 适用场景

- 拒绝率高（>30%）的场景效果更明显
- 被拒人群特征分布与通过人群差异大时
- 模型迭代时需要扩充训练样本

## GAN过采样

### 背景

信贷数据中坏样本（违约用户）通常只占3-10%，严重的类别不平衡会导致模型对少数类学习不足。

### 方法：cWGAN-based Oversampling

使用条件Wasserstein GAN（带梯度惩罚）+ 辅助分类器损失，对表格数据进行过采样。

**核心特性**：
- 支持分类变量和连续变量混合的表格数据
- 条件生成：按类别标签控制生成样本
- Wasserstein距离 + 梯度惩罚：稳定训练
- 辅助分类器：确保生成样本保持类别特征
- 目前仅支持二分类

### 适用场景

- 坏样本率极低（<5%）的场景
- 传统SMOTE在高维特征空间效果不佳时
- 需要生成更真实的合成样本时

## 其他采样方法

### SMOTE（Synthetic Minority Over-sampling Technique）

```python
from imblearn.over_sampling import SMOTE

smote = SMOTE(sampling_strategy=0.5, random_state=42, k_neighbors=5)
X_resampled, y_resampled = smote.fit_resample(X_train, y_train)
```

- 在少数类样本的特征空间中，找到K近邻
- 在样本与近邻之间的连线上随机生成新样本
- `sampling_strategy=0.5` 表示少数类达到多数类的50%

**变体**：
- **BorderlineSMOTE**：只对边界样本做过采样
- **ADASYN**：自适应合成，对难分类样本生成更多
- **SMOTE-ENN**：SMOTE + Edited Nearest Neighbors清洗

### 下采样（Undersampling）

```python
from imblearn.under_sampling import RandomUnderSampler

rus = RandomUnderSampler(sampling_strategy=0.5, random_state=42)
X_resampled, y_resampled = rus.fit_resample(X_train, y_train)
```

**注意**：会丢失多数类信息，建议配合Bagging使用。

### 金额加权采样

在信贷场景中，不同借款金额的违约损失不同。通过金额加权样本权重，让模型更关注高额度用户：

```python
# LightGBM 支持 sample_weight
sample_weights = df['loan_amount'] / df['loan_amount'].mean()
lgb_train = lgb.Dataset(X_train, y_train, weight=sample_weights)
```

在V13子分中，金额加权模型（1000w阈值）相比等权模型增益+2.6pp。

### 常用采样策略选择指南

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 坏样本率 5-10% | 不处理或轻度SMOTE | 类别不平衡不严重 |
| 坏样本率 1-5% | SMOTE + 下采样组合 | 平衡样本量和信息保留 |
| 坏样本率 <1% | GAN过采样 或 金额加权 | 需要高质量合成样本 |
| 拒绝率 >30% | 拒绝推断 | 解决选择偏差 |
| 高额度用户风险 | 金额加权 | 损失导向建模 |
