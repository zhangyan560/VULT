# 模型评估与监控方法论

> [!info]
> 本文档覆盖风控模型评估指标体系（区分能力、稳定性、业务效果三个维度）与模型监控告警体系（印尼+菲律宾双市场），并包含监控系统的技术实现与运维指南。

## 评估指标体系

### 核心评估指标

| 指标 | 业务含义 | 关注标准 | 告警阈值 |
|------|---------|---------|---------|
| **AUC** | 模型区分好坏客户的整体能力（ROC曲线面积） | 越高越好；>0.85可能过拟合 | 近7天 vs 前7天差 >±0.03 触发告警 |
| **KS** | 好坏客户分数分布的最大分离度 | 反映最优切点的拦截效率；>0.2 可用，>0.4 优秀 | 与AUC联动监控 |
| **Top Capture** | 头部N%拦截了多少坏客户 | 拒绝5%能挡住多少坏账——实际决策最关心的指标；Top5%拦截30%+为良好 | 按业务需求设定 |
| **PSI** | 模型分分布的时间稳定性（Population Stability Index） | <0.1 稳定；0.1~0.25 需关注；>0.25 必须重训 | >0.005 触发告警 |
| **CSI** | 特征维度的PSI | 输入特征的稳定性 | 同PSI |
| **分bin跳点** | 相邻分箱坏账率是否单调递增 | 非单调 → 模型在某区间排序失败 | 出现逆序分箱时人工核查 |
| **Swap分析** | 新旧模型切换时好客户被多拦截/坏客户被多放过的数量 | 直接影响业务量和坏账率，是上线决策的核心依据 | 按业务可接受的swap量设定 |
| **尾部风险** | 高分段（应该是好客户）中混入的坏客户比例 | 这些客户通过率高、额度高，造成的损失最大 | 按尾部逾期率设定阈值 |
| **覆盖率** | 特征有效覆盖的样本比例 | 覆盖率下降可能意味着数据管道故障 | 近期均值差>3%触发告警 |
| **相关系数** | 离线评估分与线上推理分的皮尔逊相关 | >0.99 表示线上复现性好；V13所有模型均>0.991 | 低于0.95需排查特征部署 |

> [!important] 最重要的指标
> 实际业务中，**Top Capture**（头部N%拦截了多少坏客户）比AUC更重要，直接决定拒绝策略效果。

### AUC指标详细说明

> [!info]
> AUC的直观理解：随机抽取一个坏客户和一个好客户，模型给坏客户打分更高的概率。AUC=0.7意味着70%的概率能正确区分。

| AUC区间 | 评级 | 业务解读 |
|---------|------|---------|
| >0.85 | 异常高 | 可能存在数据泄露（Label Echo/数据穿越） |
| 0.75~0.85 | 优秀 | 特征体系覆盖充分，或有强力子分加持 |
| 0.65~0.75 | 良好 | 印尼市场常规水平（V13主模型在此区间） |
| 0.60~0.65 | 可用 | 数据源有限时的基准水平 |
| <0.60 | 差 | 需重新设计特征或更换方法 |

### PSI指标详细说明

PSI用于衡量模型分的时间稳定性：

$$\text{PSI} = \sum_{i=1}^{n} (A_i - E_i) \times \ln\left(\frac{A_i}{E_i}\right)$$

其中$A_i$为当前分布，$E_i$为基准分布（通常为训练集分布）。

| PSI值 | 含义 | 处理建议 |
|-------|------|---------|
| <0.1 | 分布稳定 | 正常监控即可 |
| 0.1~0.25 | 分布有偏移 | 关注偏移方向，排查数据源变化 |
| >0.25 | 分布显著偏移 | 模型需重训或回滚 |

> [!important]
> 印尼市场模型衰减比国内更快，中介手法迭代、政策变化频繁。PSI告警阈值设置为0.005（比通常更严格），能更早捕捉数据分布漂移。

### Lift曲线与Top Capture

| 指标 | 定义 | V13参考值 |
|------|------|---------|
| Top5% Lift | 前5%分段的坏账率/大盘坏账率 | 通常>3.0 |
| Top10% Lift | 前10%分段的坏账率/大盘坏账率 | 通常>2.5 |
| 尾部10% Lift | 后10%分段逾期率/大盘逾期率 | 二次FPD31含埋点版2.37 |

### 线上 vs 离线一致性

| 检验项 | 方法 | V13标准 |
|--------|------|---------|
| AUC差异 | 离线AUC - 线上AUC | <0.001（V13全部通过） |
| 相关系数 | 皮尔逊相关系数 | >0.99（V13最低0.991） |
| 分布对比 | KS检验线上分与离线分 | PSI<0.05 |

### 新旧模型对比

| 指标 | 说明 |
|------|------|
| **Swap分析** | 新旧模型切换时好客户错杀/坏客户漏放数量 |
| **相关系数** | 新旧模型分数的Pearson相关系数（>0.99为稳定迁移） |
| **Lift对比** | 各分位点的Lift变化 |

> [!warning] 印尼市场特点
> 印尼市场变化快，新中介手法、政策变化导致模型衰减比国内更快，需要更频繁的监控和迭代。

## 监控告警体系

### 系统架构

该监控系统是一个海外模型监控与预警平台，主要用于监控印尼和菲律宾地区的风控模型性能。系统采用分层架构：

- **数据层**：从Hive数据仓库中提取模型预测数据和业务数据
- **处理层**：使用Python脚本进行数据处理、指标计算和告警判断
- **存储层**：将监控结果存储回Hive数据仓库
- **可视化与告警层**：支持Tableau可视化和微信告警

### 告警阈值体系

> [!warning] 告警触发条件
> 任一指标触发阈值立即发送微信群告警

| 告警类型 | 阈值 | 失效场景 |
|---------|------|---------|
| AUC告警 | 近7天 vs 前7天差 > ±0.03 | 市场环境变化或数据源异常 |
| PSI告警 | PSI > 0.005 | 输入特征分布漂移，模型适用前提被打破 |
| 覆盖率告警 | 差 > 3% | 特征缺失增加，数据管道可能故障 |
| 均值告警 | 差 > 3% | 模型分系统性偏移 |

### 指标计算频率

| 计算频率 | 指标 |
|---------|------|
| 每日 | 模型覆盖率、均值、PSI（天） |
| 每周一 | 周维度的覆盖率、均值、PSI |
| 每月1日 | 月维度的覆盖率、均值、PSI |

### 核心技术实现

**数据处理流程**：

1. **数据提取**：从Hive数据仓库中提取模型预测数据和业务数据
2. **数据转换与计算**：计算AUC、KS等模型性能指标；计算PSI值检测数据漂移；计算模型覆盖率和均值
3. **结果存储**：将计算结果存储回Hive数据仓库

**PSI计算实现**：

```python
def psi_fast(test_arr, base_arr, bins):
    test_arr = np.asarray(test_arr, dtype=float)
    base_arr = np.asarray(base_arr, dtype=float)
    if test_arr.size == 0 or base_arr.size == 0:
        return None
    bins = np.asarray(bins, dtype=float)
    if bins.size < 2:
        return None
    tcnt, _ = np.histogram(test_arr, bins=bins)
    bcnt, _ = np.histogram(base_arr, bins=bins)
    eps = 1e-6
    tprop = (tcnt + eps) / test_arr.size
    bprop = (bcnt + eps) / base_arr.size
    val = np.sum((tprop - bprop) * np.log(tprop / bprop))
    return round(float(val), 6)
```

**告警发送**：通过微信API发送告警图片至对应群组，不同类型的模型发送到不同的微信群组。

### 日常运维流程

**告警处理步骤**：

1. **接收告警**：通过微信接收告警图片
2. **分析问题**：查看告警中的指标变化，分析模型性能下降或数据漂移的原因
3. **问题处理**：联系模型开发人员进行问题排查，必要时调整模型参数或阈值
4. **记录与反馈**：记录告警处理过程，反馈处理结果到相关团队

**数据查询方式**：

```sql
-- 查询模型AUC历史数据
select * from risk_data.idn_model_auc_ks_new where model_id = '200001041';

-- 查询模型PSI历史数据
select * from risk_data.idn_model_psi_avg_cov where model_id = '200001041';
```

### 配置管理优化方案

当前 models 字典分散在各 Python 脚本中，建议分地区集中化配置管理：

- 印尼配置文件：`印尼/配置/model_config_idn.json`
- 菲律宾配置文件：`菲律宾/配置/model_config_phi.json`
- 公共配置加载器：`公共工具/config_loader.py`

新模型上线时只需修改对应地区的 JSON 配置文件，无需修改多个 Python 脚本。
