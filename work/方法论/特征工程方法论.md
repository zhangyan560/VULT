# 特征工程方法论

## 一、整体架构（8大数据源 x 7大技术路线）

特征工程的核心是将原始数据转化为模型可用的高区分力信号。体系由两个维度构成：

**8大数据源**：

| 编号 | 数据源 | 典型内容 | 特征量级 |
|------|--------|---------|---------|
| D1 | 央行征信（CBI） | 信贷记录、逾期历史、机构查询 | 万级 |
| D2 | 网贷协会（AFPI） | P2P借贷、多头借贷、在贷余额 | 万级 |
| D3 | APP安装列表 | 包名、安装/更新时间、类别 | 千级 |
| D4 | 埋点行为 | 页面浏览、点击事件、操作时长 | 百~千级 |
| D5 | 设备/地理 | 设备型号、省份、行业 | 十级 |
| D6 | 社交网络（联系人） | 紧急联系人、知识图谱关联 | 千级 |
| D7 | KYC鉴权 | 教育、收入、职业、身份验证 | 百级 |
| D8 | 多头查询（IZI等） | 跨平台查询频次、机构类型 | 百级 |

**7大技术路线**：

| 编号 | 技术路线 | 核心思想 | 适用数据源 |
|------|---------|---------|-----------|
| T1 | CTF条件聚合 | 条件(C) x 目标(T) x 聚合函数(F) 笛卡尔积 | D1, D2, D8 |
| T2 | Word2Vec/Embedding | 将离散列表映射为连续向量 | D3 |
| T3 | 图网络特征 | 二部图/相似图上的结构与风险特征 | D3, D6 |
| T4 | 社交传播特征 | 沿社交关系传播风险信号 | D6 |
| T5 | 行为统计特征 | 多时间窗口的频率/间隔/异常检测 | D4 |
| T6 | 目标编码/风险Lift | 类别变量的逾期率编码 | D5, D7 |
| T7 | 交叉特征 | 跨数据源维度组合 | D1xD3, D2xD3, D7xD8 |

```
数据源层（8大源）
    |
    v
特征衍生层（7大技术路线）
    |
    v
特征筛选层（PSI稳定性 -> IV/AUC效果 -> WOE评估 -> VLM监控）
    |
    v
模型融合层（子分模型 -> 增益评估 -> 上线部署）
```

## 二、CTF条件聚合框架

### 2.1 核心思想

CTF（Condition-Target-Function）是征信类结构化数据特征挖掘的通用范式。对用户的多笔订单记录，按三维笛卡尔积系统生成特征：

- **C（Condition）**：过滤条件，如时间窗口、机构类型、逾期状态
- **T（Target）**：聚合目标，如订单数、在贷金额、逾期天数
- **F（Function）**：聚合函数，如 sum / mean / max / min / std / count

### 2.2 多维度交叉体系

**条件维度设计模板**（以征信数据为例）：

| 条件组 | 条件列示例 | 取值示例 |
|--------|-----------|---------|
| 时间窗口 | gap_days_start_bin | betw180and360, ge720 |
| 机构类型 | inst_scale_type, inst_risk_type | top_inst, tail_inst, high_risk |
| 逾期状态 | pastdue_days_bin | Current, DPD1-89, DPD180+ |
| 合同状态 | contract_status_type | ActiveFacilities, Paidoff, Writtenoff |
| 金额分段 | initial_creditlimit_bin | le3w, betw3wand10w, ge500w |
| 期限分段 | loan_term_bin | le30, betw30and90, ge720 |

**阶数机制**：

```
0阶：无条件全量聚合
  命名：num_order, sum_outstanding

1阶：单条件过滤
  命名：inst_scale_type=top_inst#num_order

2阶：双条件交叉（AND逻辑）
  命名：gap_days_start_bin=betw180and360&inst_scale_type=top_inst#sum_outstanding
```

**跨窗口趋势特征**：

在 3M / 6M / 12M / 24M / 全量 五个时间窗口分别计算同一指标，然后衍生窗口间差值和比值：

```python
# 窗口间趋势特征
trend_diff_3m_6m = feature_3m - feature_6m    # 短期变化
trend_ratio_3m_12m = feature_3m / feature_12m  # 短期占长期比
```

### 2.3 征信数据条件聚合模板

**CBI条件聚合示例**：

```sql
-- 1阶特征：近6个月头部机构的在贷余额均值
SELECT loan_account_id,
       AVG(debit_balance) AS inst_scale_type_top_inst__mean_outstanding_w6m
FROM cbi_orders
WHERE gap_days_start <= 180
  AND inst_scale_type = 'top_inst'
GROUP BY loan_account_id;

-- 2阶特征：近12个月高风险机构且有逾期的订单数
SELECT loan_account_id,
       COUNT(*) AS inst_risk_type_high_risk__pastdue_gt0__num_order_w12m
FROM cbi_orders
WHERE gap_days_start <= 360
  AND inst_risk_type = 'high_risk'
  AND pastdue_days > 0
GROUP BY loan_account_id;
```

**AFPI条件聚合模板**：

```sql
-- 按借款金额等级 x 时间窗口聚合
SELECT loan_account_id,
       SUM(CASE WHEN amount_bin = 'ge300w' AND window = '90d'
           THEN loan_amount ELSE 0 END) AS amount_ge300w_w90d_sum_loan_amount,
       COUNT(CASE WHEN status = 'overdue' AND window = '180d'
           THEN 1 END) AS status_overdue_w180d_num_order
FROM afpi_orders
GROUP BY loan_account_id;
```

## 三、目标编码（Target Encoding）

### 3.1 原理

将高基数类别变量（如设备型号、省份、APP包名）编码为该类别的目标变量均值（如逾期率）。本质是用先验知识替代独热编码，减少维度爆炸。

### 3.2 贝叶斯平滑公式

直接使用原始逾期率在小样本类别上不稳定。贝叶斯平滑将小样本类别向全局均值收缩：

```
smoothed_rate = (n_bad + overall_rate * C) / (n_total + C)
```

- `n_bad`：该类别的坏样本数
- `n_total`：该类别的总样本数
- `overall_rate`：全局坏样本率
- `C`：平滑因子，推荐值50（样本量越大，C可适当增大）

**直觉**：当 `n_total` 远大于 C 时，结果趋近于该类别的真实逾期率；当 `n_total` 远小于 C 时，结果趋近于全局均值。

### 3.3 Lift编码

在贝叶斯平滑基础上，用相对值替代绝对值：

```
lift = smoothed_rate / overall_rate
```

- Lift > 1.0：高于平均风险
- Lift < 1.0：低于平均风险

输出示例：`device_mob1_label7_lift`、`province_mob3_label7_lift`

### 3.4 防泄漏：K-Fold Target Encoding

在训练集中直接计算TE会导致信息泄漏。标准做法：

```python
from sklearn.model_selection import KFold

kf = KFold(n_splits=5, shuffle=True, random_state=42)
te_col = np.zeros(len(train))

for train_idx, val_idx in kf.split(train):
    # 仅用train_idx计算编码映射
    stats = train.iloc[train_idx].groupby(cat_col)[target].agg(['sum', 'count'])
    stats['te'] = (stats['sum'] + overall_rate * C) / (stats['count'] + C)
    mapping = stats['te'].to_dict()
    # 应用到val_idx
    te_col[val_idx] = train.iloc[val_idx][cat_col].map(mapping)
```

**OOT集处理**：使用训练集全量数据计算的编码映射，不使用OOT自身信息。

### 3.5 稳定性验证

- 计算各类别逾期率的月度变异系数（CV）
- CV > 0.3 标记为不稳定，CV > 1.0 标记为极不稳定
- 不稳定类别单独分组或回退到全局均值

## 四、App图网络特征

App图网络基于"安装了相似App的用户往往有相似的风险特征"这一假设，构建用户-App二部图后提取多层特征。

**核心方法要点**：

- **二部图构建**：用户节点 - App节点，过滤超级App（安装率>75%）和长尾App（用户数<5）
- **贝叶斯平滑节点属性**：`app_risk = (bad + overall_rate * 50) / (total + 50)`
- **边属性**：Jaccard相似度、Risk Lift（共现坏账率/全局坏账率）
- **Top-K裁剪**：每个App只保留Top20邻居，存储成本降至<3%
- **PIT时间切片**：严格按日期构建独立快照图，防止未来信息泄露
- **LOO保护**：App风险计算排除当前批次，buffer 30-60天
- **双层架构**：Behavior Graph（无标签纯共现）+ Risk Projection Layer（轻标签带LOO）

**6层特征体系**：覆盖度(L0) -> 节点风险(L1) -> 二跳传播(L2) -> 边级风险(L3) -> 社区检测(L4) -> 子图拓扑/SVD嵌入(L5-L6)

详细方法论与生产部署方案参见 [[App图网络特征衍生框架_生产版]]。

## 五、社交网络特征

### 5.1 关联关系定义

| 关系类型 | 定义 | 数据来源 | 风险含义 |
|---------|------|---------|---------|
| 一度关系 | 用户A的紧急联系人 = 用户B | 紧急联系人电话号码匹配 | 直接社交圈风险传播 |
| 二度关系 | 一度联系人的联系人 | 递归查询 | 扩展社交圈风险 |
| 弱关联 | 同一手机号关联多账户 / 同一紧急联系人手机号 | 知识图谱推导 | 团伙欺诈信号 |
| 设备关联 | 同一设备ID登录的不同用户 | 设备指纹匹配 | 设备农场/代办中介 |

### 5.2 社交传播特征设计思路

**核心假设**：一个人的紧急联系人如果多次出现在逾期用户的关系网中，该用户的违约概率也更高。

**传播内容**：
- 风控子模型评分（多个模型分 x 方向 x 联系人类型 x 时间窗口 x 聚合方式）
- 过件额度与评级（信用额度、评级占比）
- 征信变量（CBI/AFPI核心指标的邻居聚合）

**维度交叉模板**：

```
{方向} x {联系人类型} x {传播内容} x {聚合方式} x {时间窗口}

方向：in（被列为联系人）/ out（列他人为联系人）/ weak（弱关联）
联系人类型：immediate（第一联系人）/ second_immediate（第二联系人）
聚合方式：AVG / MAX / MIN
时间窗口：7d / 30d / 90d / 180d / all
```

命名示例：`in_immediate_modelid_200001311_avg_7d`

### 5.3 设备共享特征

通过设备ID关联不同用户，提取：
- 邻居人口统计聚合：`avg/max/min_{age, monthly_income, mob1_pd7}`
- 设备属性：`avg_device_{memory, storage, pagerank}`
- 图中心性：PageRank（alpha=0.85）

### 5.4 防泄漏

社交传播特征严格遵循时间约束：`first_done_ts(当前用户) > first_done_ts(邻居用户)`，确保只使用已经成熟的邻居标签信息。

## 六、行为统计特征（埋点）

### 6.1 时间窗口体系（累积包含原则）

埋点特征使用7个时间窗口，较长窗口包含所有较短窗口的事件：

```
1h < 3h < 6h < 12h < 1d < 3d < 7d

示例：switch_out_cnt_3h 包含 1h 和 3h 窗口内的所有切出事件
```

**时间窗口的业务含义**：
- 1h/3h：捕捉"申请当下"的异常行为
- 6h/12h：捕捉"申请前后"的准备行为
- 1d/3d/7d：建立用户常规使用习惯基线

### 6.2 异常检测模式（短时高频）

检测短时间桶内的异常高频操作，用于识别机器人、脚本或远程控制：

```sql
-- 异常频率检测：将时间轴切分为固定桶（如10s/1min/5min）
-- 统计每个桶内的事件数，取最大值/均值/标准差
SELECT loan_account_id,
       MAX(bucket_count) AS max_events_per_10s_7d,
       AVG(bucket_count) AS avg_events_per_10s_7d,
       STDDEV(bucket_count) AS std_events_per_10s_7d
FROM (
    SELECT loan_account_id,
           FLOOR(UNIX_TIMESTAMP(event_time) / 10) AS bucket_id,
           COUNT(*) AS bucket_count
    FROM events
    WHERE time_window = '7d'
    GROUP BY loan_account_id, bucket_id
) bucketed
GROUP BY loan_account_id;
```

**生产实现**：使用 shift 邻桶检测代替 GROUP BY 动态桶ID，避免Kite算子的跨算子引用限制。

### 6.3 反欺诈信号设计思路

| 行为信号 | 风控含义 | 特征设计 |
|---------|---------|---------|
| 频繁切出/切入App | 中介同时操控多台设备 | switch_out_cnt / switch_in_cnt per window |
| 短时高频操作 | 脚本/机器人行为 | max_events_per_{10s/1min/5min} |
| 截屏事件 | 信息窃取 | screenshot_cnt per window |
| 夜间操作比例 | 中介集中晚间批量操作 | night_ratio = night_events / total_events |
| 填表速度过快 | 中介操作熟练 | auth_step_duration_min |
| IP/设备频繁切换 | VPN/代理/设备农场 | distinct_ip_count, ip_switch_count |

### 6.4 其他模式

- **页面分布统计**：各页面停留时长占比，高风险页面访问频率
- **时间间隔统计**：LEAD()获取下一事件时间，计算avg/max/min间隔
- **时间分布统计**：夜间/白天/工作日/周末的行为分布
- **活跃天数**：连续活跃天数、日均操作频率
- **会话特征**：会话数、平均时长、短会话/长会话占比

## 七、Word2Vec/Embedding

### 7.1 APP列表向量化

将每个用户的APP安装列表视为一个"句子"，每个APP包名视为一个"词"，使用Word2Vec学习APP的分布式表示：

**训练流程**：

```python
from gensim.models import Word2Vec

# 1. 构建语料：每个用户的APP列表按安装时间排序
sentences = []
for user in users:
    apps = sorted(user.apps, key=lambda x: x.install_time)
    apps = [a.package_name for a in apps if not a.is_system_app]
    sentences.append(apps)

# 2. 训练Skip-gram模型
model = Word2Vec(
    sentences,
    vector_size=64,     # 嵌入维度
    window=8,           # 上下文窗口
    sg=1,               # Skip-gram
    negative=15,        # 负采样数
    min_count=10,       # 最小出现次数
    epochs=10
)

# 3. 用户向量聚合
user_vector = np.mean([model.wv[app] for app in user_apps if app in model.wv], axis=0)
```

### 7.2 APP风险分级（前置目标编码）

在Embedding聚合之前，先对每个APP按逾期率分级（12个风险等级），然后在每个风险等级内分别聚合Embedding：

```
特征维度 = 风险等级(12) x 时间窗口(3: all/last180/last30) x 聚合方式(3: mean/max/sum) x 向量维度(64)
```

命名模式：`{level}_{window}_{agg}_index{i}`，如 `level_5_last180_mean_index32`

### 7.3 聚类与相似度特征

- KMeans聚类：对Embedding做k=5聚类，输出用户所属簇
- 余弦相似度：用户向量与"逾期用户中心向量"的余弦距离

## 八、特征筛选管线

### 8.1 标准筛选流程

```
全量候选特征
    |
    v
Step 1: 缺失率筛选（缺失率 < 98%）
    |
    v
Step 2: PSI稳定性筛选
    - 计算训练集 vs OOT的PSI
    - PSI > 0.25 删除（分布严重漂移）
    - PSI > 0.10 标记待审核
    |
    v
Step 3: LGB重要性初筛
    - 训练包含全量变量的LightGBM
    - 保留 importance_percentage > 0 的特征
    |
    v
Step 4: 单特征效果筛选
    - IV > 0.02 或 单特征AUC > 0.52
    |
    v
Step 5: WOE稳定性评估
    - 训练集WOE分箱 -> OOT集评估
    - 计算Kendall相关系数
    - corr <= 0: 不稳定，删除
    - 0 < corr <= 0.5: 临界，人工审核WOE趋势图
    - corr > 0.5: 稳定，保留
    |
    v
Step 6: VLM分布监控（线上特征分布漂移检测）
    |
    v
最终入选特征
```

### 8.2 管线参数参考

以实际大版本迭代为参考，从79,801个候选特征到19,479个入选特征的筛选路径：

| 数据源 | 候选特征 | 稳定性筛后 | 效果筛后（最终） |
|--------|---------|-----------|---------------|
| APP | 33,063 | 16,369 | 4,879 |
| AFPI | 12,674 | 10,912 | 5,873 |
| CBI | 23,949 | 15,494 | 5,240 |
| IZI | 8,641 | 5,841 | 2,109 |
| 鉴权 | 515 | 469 | 469 |
| ADVANCE | 315 | 265 | 265 |
| 同盾 | 644 | 644 | 644 |
| **合计** | **79,801** | — | **19,479** |

**关键观察**：APP和CBI的稳定性淘汰率最高（>50%），因为这两类数据源的分布变化最大；鉴权和同盾特征稳定性最好，几乎全量通过。

### 8.3 WOE评估详细方法

```python
# WOE分箱评估
from sklearn.metrics import roc_auc_score
import scipy.stats as stats

def evaluate_woe_stability(train_woe, oot_woe, train_y, oot_y):
    """评估WOE在OOT上的稳定性"""
    # 1. 训练集分箱
    bins = optimal_binning(train_woe, train_y, max_bins=10)

    # 2. 计算训练集各bin的WOE值
    train_woe_values = calc_woe_per_bin(train_woe, train_y, bins)

    # 3. 将同一分箱应用到OOT
    oot_woe_values = calc_woe_per_bin(oot_woe, oot_y, bins)

    # 4. Kendall相关系数
    tau, p_value = stats.kendalltau(train_woe_values, oot_woe_values)

    # 判定
    if tau <= 0:
        return 'UNSTABLE'   # 排序完全反转
    elif tau <= 0.5:
        return 'MARGINAL'   # 需人工审核
    else:
        return 'STABLE'     # 通过
```

## 九、PIT防泄漏原则

### 9.1 时间切片（Point-in-Time）

所有特征计算必须满足：**计算某用户特征时，只能使用该用户进件时点之前已成熟的数据**。

```
时间轴：
   past ←──────────────────────────────→ future

   |-- 可用数据 --|-- 缓冲期 --|-- 用户进件 --|-- 标签成熟 --|
                               ^
                          feature cutoff
```

### 9.2 LOO Buffer（Leave-One-Out缓冲）

在图网络和目标编码中，避免标签回声（Label Echo）：

- **App图网络**：计算App风险评分时排除当前批次（30天内进件的用户），buffer推荐30-60天
- **社交传播**：邻居聚合只使用 `first_done_ts < 当前用户first_done_ts` 的已成熟邻居
- **Target Encoding**：K-Fold TE确保训练时每个样本不使用自身标签

### 9.3 标签不参与拓扑

在图网络的双层架构中：

- **Behavior Graph（行为图）**：完全无标签，仅使用APP共现、安装时间等行为信息构建图结构
- **Risk Projection Layer（风险投影层）**：使用已成熟标签计算节点风险属性，但标签不影响图的边连接关系

### 9.4 OOT严格隔离

- 训练集/OOT按 `payout_date` 或 `first_done_date` 切分
- Target Encoding / WOE 统计仅使用训练期数据
- Word2Vec训练语料不包含OOT期间新增用户
- 自身APP排除：嵌入计算排除自家产品APP

## 十、征信-APP联动特征

### 10.1 核心思路

将APP安装状态与征信借款记录交叉，捕捉安装行为与借贷行为之间的一致性/矛盾性信号。

### 10.2 交叉矩阵

| App安装状态 | 有该机构借款 | 曾经逾期 | 是否结清 | 衍生指标 |
|------------|-----------|---------|---------|---------|
| 借前已安装 | 是 | 是/否 | 是/否 | 逾期金额/天数/平均额度/借款次数/机构数/时间间隔 |
| 借后180天内安装 | 是 | 是/否 | / | 同上 |
| 借后180天后安装 | 是 | 是/否 | / | 同上 |
| 未安装 | 是 | 是/否 | / | 逾期金额/天数/额度/次数/机构数（卸载行为信号） |
| 已安装 | 否 | / | / | 安装App个数（探索性安装行为） |

### 10.3 命名模式

```
{数据源}_APP_DATA_FEATURE___installStatus{Status}___w{Window}___{Metric}___{Stat}

维度取值：
- installStatus: InstallBefore / InstallAfter180d / InstallIn180d / NotInstalled
- Window: w3m / w6m / w12m / w24m / wall
- Metric（AFPI）: tLoanAmount / tRepaymentRatio / creditDayPastDue / ...
- Metric（CBI）: creditLiqCurrentMonth / creditInterestRate / creditPenalty / ...
- Stat: mean / sum / max / min / std / cv / count
```

### 10.4 设计直觉

- **装了App但没借款**：探索性安装，可能在比较多个平台
- **借了款但卸了App**：可能有逃避催收意图
- **借前安装且未逾期**：正常使用路径
- **未安装但有借款记录**：可能通过非APP渠道借款，或已卸载

## 十一、借新还旧特征

### 11.1 核心目的

检测跨平台"拆东墙补西墙"行为——用户在A平台即将到期/已逾期时向B平台借新款偿还。

### 11.2 四维度组合设计

| 维度 | 可选值 | 说明 |
|------|-------|------|
| 对比偏移(shift) | 前第1/2/3/4笔订单 | 按CREDIT_CreditStartDate排序 |
| 时间差窗口 | 前14/60/120天内 | 新订单开始日期在旧订单到期日前/后N天 |
| 筛选条件 | prev_closed / prev_overdue / amount_increased / creditor_changed | 上一笔结清？逾期？金额增？新机构？ |
| 时间维度 | 6/12/24/all月 | 历史回看窗口 |

### 11.3 命名模式

```
# 基础借新还旧标记
rollover_before_{window}_{shift}    # 旧订单到期前N天内借新
rollover_after_{window}_{shift}     # 旧订单到期后N天内借新

# 条件组合
*prev_closed                        # 前一笔已结清
*prev_overdue                       # 前一笔有逾期
*amount_increased                   # 借款金额增加
*creditor_changed                   # 更换了债权机构

# 订单间时间差统计
days_between_orders_{type}_{shift}_{stat}_{window}
  type: ss(开始-开始) / sd(开始-到期) / dd(到期-到期)
  stat: mean / max / min / std / count

# 条件筛选后的时间差统计
{condition}_conditional_days_{type}_{shift}_{stat}_{window}
```

### 11.4 高价值特征模式

实践中IV值最高的特征集中在：**条件时间差类**（IV约0.015-0.020），尤其是前1-4笔已结清订单的开始/到期日期差的统计量。这反映了用户在短时间内密集借还的"以贷养贷"行为模式。
