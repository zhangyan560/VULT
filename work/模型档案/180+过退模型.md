# 180+过退模型

> [!info]
> 180+过退模型针对被拒绝超过180天后重新申请的特殊客群，这类客群随着买量策略开放而增加，需要专项模型处理。

## 业务背景

| 项目 | 详情 |
|------|------|
| 背景 | 180+天拒绝重审客群开启买量，历史被拒用户重新进入申请流程 |
| 客群特点 | 曾被风控系统拒绝，风险特征与常规客群不同；距离上次申请已超过180天，状态可能已变化 |
| 目标 | 预测一次风控通过率，从被拒客群中筛选高通过率的可放款人群 |
| 建模方法 | LightGBM |

## 核心效果指标

| 指标 | 数值 | 对比基准 |
|------|------|---------|
| AUC提升（vs V11前筛） | +6pp | V11前筛 |
| AUC提升（vs V10） | +14pp | V10版本 |
| bin10通过率 | 63% | — |
| V11 bin10通过率 | 46% | 提升17% |

> [!important]
> 相比V11前筛AUC提升+6pp，相比V10提升+14pp，说明针对该特殊客群的专项建模效果显著。bin10（最高分档）通过率从46%提升至63%，每次放款精准度大幅提升。

## 客群分层设计

按距离上次拒绝时间（day_diff）分层，不同时间段的用户风险特征差异明显：

| 分层 | day_diff范围 | 风险特征 |
|------|------------|---------|
| 极短间隔 | <30天 | 风险最高，刚被拒绝即重申 |
| 短间隔 | 30~60天 | 较高风险 |
| 中间隔 | 60~120天 | 中等风险，部分用户状况已改善 |
| 较长间隔 | 120~180天 | 风险相对较低 |
| 超长间隔 | >180天 | 风险最低，时间距离越长用户状态变化越大 |

> [!info]
> 按day_diff分层是该模型的核心设计之一：距离上次被拒越久，用户经济状况改变的概率越高，简单套用主评级分层标准不适用，需要专项分层策略。

## 建模方法

- **算法**：LightGBM梯度提升树
- **标签**：一次风控通过/拒绝（二分类）
- **训练样本**：历史180+天后重申客群
- **特征**：外部征信数据（CBI/AFPI/IZI）+ 设备特征 + 历史申请记录
