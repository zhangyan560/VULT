# 优选贷发标模型体系

> 上级页面：[[信也优选贷业务]]

## 业务场景

**发标模型**作用于用户发标（确认借款）环节，是信贷决策流程的最终风控关卡。

主要功能：
1. **拒绝**：对高风险用户直接拒绝，不给予放款
2. **拆标**：对中等风险用户降低发标金额（拆标金额 = 降额系数 × 可用额度）

发标模型位于主模型和评级之后，针对已通过主模型初筛的人群进行二次精细筛查，重点识别**尾部高风险人群**，尤其是**高额短期内逾期的坏人**。

### 发标流程说明（基于20230913文档）

```
发标页1: 目标人群输入拆标金额 → 对发标金额 > 拆标金额的人群进行拆标
         目标人群（当前）: B评级 额度10w以下

发标页2: 未拆标 → 走发标模型拒绝
         拆标后可发金额 → 走发标模型拒绝
         > 拆标金额部分 → 额外拒绝策略（75%的标会通过）
```

**阈值切法**：
- 按老模型对标大致拒绝比例，调整新模型拒绝阈值，保持预审通过率持平
- 根据成交和风险变化，前后移动阈值寻找最佳决策点
- 根据整体风险大环境，激进或保守调整拒绝阈值

## 各版本关键对比

### 版本间主要变化汇总

| 维度 | y5 → y6 | y6 → y7 |
|------|---------|---------|
| 背景 | y5上线近8个月，在发标评级迭代前预先迭代 | 字节下线，需紧急迭代补充损失 |
| Target | target_ever30_3m | target_ever_3m_30（一致） |
| 样本 | 2021-06之后优选贷成交用户 | 2022-10至2023-10（ins），OOT延至2024-03 |
| 新增变量类型 | UBT变量、新三方数据源 | APP变量+深度学习分、UBT23年版、腾讯v11xgb、同盾物流联合建模、天创联合建模 |
| 移除变量 | - | 腾讯V9、字节联合建模、友盟V6、百融联合建模、银联v2 |

## 模型架构说明

发标模型在整体流程中的位置：

```
戳额 → 数据源调用 → 分流模型 → 主模型打分（20bin）
     → 规则分（5bin）→ 二维评级 → 额度分（7bin）→ 三维评级
     → 发标流程 → [发标模型 拒绝/拆标] → 最终授信决策
```

**与主模型的关系**：
- 主模型迭代周期约6个月，cap规则：`cap_bin = max(old_bin-4, new_bin)`
- 发标模型仅用于拒绝时，可不考虑业务指标和金额排序，仅关注尾部足够黑
- Cross主模型 × 发标模型可检验两者协同效果
