# 优选贷发标模型y6

> 上级页面：[[发标模型体系]] | 源文件：`20230814_优选贷发标模型y6_张琰.xlsx`

## 模型概述

| 属性 | 内容 |
|------|------|
| 建模时间 | 2023年8月（文件名：20230814） |
| 建模人 | 张琰 |
| 上线时机 | 2023年9月发标评级迭代前 |
| 主要目标 | 提升对尾部高额短逾坏人的识别能力 |
| 算法 | LightGBM |
| 作用 | 发标环节拒绝（不用于拆标） |

**背景**：
1. y3、y4策略已下线，y5上线近8个月，在9月份发标评级迭代之前，预先迭代一版新的发标模型
2. 旧版发标模型（y5）尾部对高额坏人识别能力较差，尤其是高额短期内逾期的人
3. 该版发标模型仅用于拒绝，因此可不考虑业务指标和金额排序，仅关注尾部足够黑

**结论摘要**：
1. 使用常规变量 + 发标变量 + UBT变量，新接入了百度v3、字节分等三方数据源
2. 使用较短期target（target_ever30_3m）中和期限变量的过度影响
3. 采取对高额短逾坏人加大权重的方式（fb_v1~v4）提升高额坏人识别
4. 推荐方案fb_v1（不加权版本）：拒绝相同金额情况下，拒绝风险约为y5的**1.43倍**，拒绝人数约为y5的**59%**
5. 人头等切时：拒绝平均金额为y5的**2倍以上**，拒绝风险约为y5的**1.2倍**

## 模型方案

| 方案 | 描述 | 权重设计 |
|------|------|---------|
| fb_v1 | 常规建模，不加权 | 无权重 |
| fb_v2 | 高额短逾加权（中等力度） | 金额>2万: 1.3; >3万: 1.5; >5万: 2 |
| fb_v3 | 高额短逾加权（较强力度） | 金额>2万: 1.5; >3万: 2; >5万: 3 |
| fb_v4 | 高额短逾加权（强力度） | 金额>2万: 2; >3万: 3; >5万: 4; >7万: 5 |

**最终推荐方案**：fb_v1（加权会导致金额倒序，尾部拒绝风险不如不加权版本）

## 与y5的版本对比

| 维度 | y5 | y6 (fb_v1) |

| 拒绝风险（金额等切） | 基准 | 约y5的 **1.43倍** |

| UBT变量 | 无/少 | 新增UBT页面点击类变量 |
| 新三方数据源 | 无 | 百度v3、字节分（y7中下线） |
