# 图像子分模型

> [!info]
> 图像子分模型基于用户上传的人脸照片进行风险判别，作为传统结构化特征的补充，捕捉图像中蕴含的风险信号。

## 模型架构

| 项目 | 详情 |
|------|------|
| 基础架构 | ResNet18（残差网络18层） |
| 输入形式 | 多图输入（用户上传的多张人脸/证件照） |
| 任务类型 | 二分类（好客户/坏客户） |
| 特征来源 | 人脸照片（申请时上传） |

> [!important]
> 多图输入设计：用户上传多张照片（如正面人脸+证件照+手持证件照），ResNet18分别提取各图特征后融合，提升对单图质量问题的鲁棒性。

## 业务逻辑

> [!info]
> 图像子分的核心假设：图像中包含传统结构化特征无法捕捉的风险信号。例如：照片质量异常（代办中介批量使用模糊/盗用照片）、人脸图像的某些统计特征与历史逾期用户存在相关性。

| 信号类型 | 业务含义 |
|---------|---------|
| 照片质量 | 代办中介批量操作时照片质量参差不齐，或使用他人照片 |
| 人脸特征 | 图像内容中的隐含统计特征与风险的相关性 |
| 多图一致性 | 多张照片的人脸一致性，识别使用他人证件的欺诈行为 |

## 在子分体系中的位置

图像子分作为V13子分体系的一部分，属于反欺诈方向子分：
- 与主评级的相关性较低（捕捉主模型遗漏的图像维度信息）
- 作为融合层的子分输入，与其他结构化子分一起参与融合

## 训练数据

| 项目 | 详情 |
|------|------|
| 数据来源 | 用户申请时上传的人脸照片 |
| 标签 | MOB1 7+逾期（主Y值，与主评级一致） |
| 存储 | 图片数据存储于业务系统，按loan_account_id关联 |

## 技术要点

- **预训练权重**：使用ImageNet预训练的ResNet18，微调（fine-tune）于风控任务
- **多图融合**：各图特征向量在全连接层之前进行平均池化或注意力融合
- **类别不平衡**：坏样本率通常<10%，需使用类别权重或上采样处理
