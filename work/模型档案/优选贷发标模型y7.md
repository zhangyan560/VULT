# 优选贷发标模型y7

> 上级页面：[[发标模型体系]] | 源文件：`20240528_发标模型y7_张琰.xlsx`

## 模型概述

| 属性 | 内容 |
|------|------|
| 建模时间 | 2024年5月（文件名：20240528） |
| 建模人 | 张琰 |
| 状态 | 当前线上版本 |
| 背景触发 | 字节渠道下线，需紧急迭代补充损失 |
| 算法 | LightGBM |
| 作用 | 发标环节拒绝 |

**背景**：
1. 为应对字节下线，需紧急迭代优选贷发标模型
2. 同时有新的数据源接入（腾讯v11xgb、同盾物流联合建模、天创联合建模），可弥补字节下线带来的损失

**结论摘要**：
1. y7相对y6，AUC提升**3个点以上**，尾部风险倍数为y6的**1.3倍**，尾部金额为y6的**1.2倍**
2. 保留了期限变量，建议分长短期限使用不同阈值进行拒绝（12期 vs <12期）
3. y7_12q（仅12期样本建模）和y7_cul30（剔除30天内提前结清）在12期人群上效果与y7接近，但在<12期人群上效果远不如y7
4. Cross主模型后，y7尾部在主模型头部命中的人群要比y6风险更高

## 分期限策略建议

y7 Excel中特别指出保留了期限变量，且建议：
- **12期人群**：使用一个拒绝阈值
- **<12期人群**：使用另一个（更宽松或更严格的）拒绝阈值

原因：y7_12qi（仅12期建模）在<12期人群上效果远不如y7，说明期限对风险识别有显著调节作用，不宜在全量人群上用同一阈值。
