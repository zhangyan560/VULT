## 三、统一代码架构（首贷 + 复贷）

> **核心设计**：特征表以 **`loan_account_id × dt`** 为粒度生产，与 `trace_id` 完全解耦。  
> 训练/评分时，通过 `JOIN ON loan_account_id AND dt = anchor_date` 做时间切片回溯，天然支持四个场景复用同一张特征表。

### 3.1 整体分层设计

```
┌──────────────────────────────────────────────────────────────────┐
│                    埋点 T+1 特征生产层（每日刷新）                    │
│                                                                  │
│  Source:  ec_dwd.dwd_ec_log_di_sa_app_events                    │
│               ↓ 每日增量追加（仅 70 个目标事件）                     │
│  中间层:  yanzhang1.yz_event3_t1_daily_cnt                        │
│           grain: loan_account_id × event_code × event_date       │
│               ↓ 回溯 360d 聚合，每日生成一个分区                     │
│  特征表:  yanzhang1.yz_event3_t1_features                         │
│           grain: loan_account_id × dt  ← 核心粒度                │
│           partition: dt (YYYY-MM-DD)                             │
└──────────────────────────────────────────────────────────────────┘
                                │
                   ┌────────────┴─────────────┐
                   ▼                          ▼
    ┌──────────────────────┐    ┌───────────────────────────────┐
    │   在线评分（实时）     │    │   离线训练（按时间切片回溯）     │
    │                      │    │                               │
    │  SELECT f.*          │    │  SELECT b.trace_id,           │
    │  FROM features f     │    │         b.label, f.*          │
    │  WHERE               │    │  FROM   base_table b          │
    │    loan_account_id=X │    │  JOIN   features f            │
    │    AND dt = today-1  │    │    ON   b.loan_account_id     │
    │                      │    │       = f.loan_account_id     │
    │                      │    │    AND f.dt = b.anchor_date   │
    └──────────────────────┘    └───────────────────────────────┘
```

### 3.2 "特征生产"与"特征取用"分离

| 层 | 粒度 | trace_id | 说明 |
|----|------|----------|------|
| **特征生产**（daily task） | `loan_account_id × dt` | ❌ 无 | 每天为所有活跃用户预计算 1,470 列特征 |
| **特征取用**（训练/评分） | 按 `anchor_date = dt` 切片 JOIN | ✅ 仅在 base_table 侧 | `trace_id` 只用于拼接标签，不进入特征表 |

**好处**：
- 同一张特征表服务全部四个场景（首贷一次/二次、复贷一次/二次）
- 支持任意时间点回溯，无需按 `trace_id` 重复计算
- 防穿越由特征生产侧保证（`event_date <= dt`）

### 3.3 关键 SQL 模板

**特征生产（T+1 每日写入，`loan_account_id × dt` 粒度）**

```sql
-- 【Step1】每日增量追加中间层（append-only，partition by event_date）
INSERT OVERWRITE TABLE yanzhang1.yz_event3_t1_daily_cnt
    PARTITION (event_date = '${dt}')
SELECT
    loan_account_id,
    event_code,
    COUNT(*) AS daily_cnt
FROM ec_dwd.dwd_ec_log_di_sa_app_events
WHERE dt     = REGEXP_REPLACE('${dt}', '-', '')  -- YYYYMMDD 分区
  AND event_code IN (${event_list})
  AND loan_account_id IS NOT NULL
GROUP BY loan_account_id, event_code;

-- 【Step2】生成特征宽表分区（回溯 360d，GROUP BY loan_account_id）
INSERT OVERWRITE TABLE yanzhang1.yz_event3_t1_features
    PARTITION (dt = '${dt}')
SELECT
    loan_account_id,
    -- cnt 系列（9 个窗口）
    SUM(CASE WHEN event_code='{E}' AND datediff('${dt}', event_date) <   1 THEN daily_cnt ELSE 0 END) AS {E}_cnt_1d,
    SUM(CASE WHEN event_code='{E}' AND datediff('${dt}', event_date) <   7 THEN daily_cnt ELSE 0 END) AS {E}_cnt_7d,
    SUM(CASE WHEN event_code='{E}' AND datediff('${dt}', event_date) <  30 THEN daily_cnt ELSE 0 END) AS {E}_cnt_30d,
    SUM(CASE WHEN event_code='{E}' AND datediff('${dt}', event_date) <  90 THEN daily_cnt ELSE 0 END) AS {E}_cnt_90d,
    SUM(CASE WHEN event_code='{E}' AND datediff('${dt}', event_date) < 360 THEN daily_cnt ELSE 0 END) AS {E}_cnt_360d,
    -- ever / days / max / recency ...（其余特征同理）
    MIN(CASE WHEN event_code='{E}' THEN datediff('${dt}', event_date) END) AS {E}_recency
FROM yanzhang1.yz_event3_t1_daily_cnt
WHERE event_date <= '${dt}'
  AND event_date >= DATE_SUB('${dt}', 360)
GROUP BY loan_account_id;   -- ← 粒度：loan_account_id × dt（dt 已在 PARTITION 中）
```

**特征取用（训练时按 anchor_date 时间切片回溯，无穿越）**

```sql
-- 四个场景通用：只需改 base_table，特征表完全复用
-- anchor_date 决定取哪一天的特征分区（= 风控时刻的前一天）
SELECT
    b.trace_id,
    b.loan_account_id,
    b.anchor_date,
    b.mob1_dpd7_ind       AS label,
    b.mob1_dpd7_matured,
    f.*                   -- 1,470 列特征，无穿越（均为 anchor_date 前的历史数据）
FROM {BASE_TABLE} b
JOIN yanzhang1.yz_event3_t1_features f
    ON  b.loan_account_id = f.loan_account_id
    AND f.dt              = b.anchor_date   -- ← 时间切片：取对应日期特征分区
WHERE b.mob1_dpd7_matured = 1;
```

### 4.3 七维度特征展开

> **命名规则**：特征前缀 = 原始 event_code **去掉所有下划线**，完整特征名形如 `{event_code去掉下划线}_{类型}_{窗口}`。
> 例：`loan_b1_pageview` → 特征前缀 `loanb1pageview`，特征名 `loanb1pageview_cnt_7d`。
> 各维度以代表性事件展示特征名示例，其余 64 个事件按同等模式扩展。

#### 维度一：基础聚合特征（cnt / days / ever / max_daily）

**设计思路**

```
对每个事件 E，在每个时间窗口 W 下，生成四类基础特征：
  cnt     → 窗口内事件发生总次数
  days    → 窗口内有该事件的天数（≤ W）
  ever    → 窗口内是否至少发生1次（0/1）
  max_daily → 窗口内单天最大发生次数
```

**模板定义**

| 特征类型 | 命名规则 | SQL 函数 |
|---------|---------|---------|
| 累计次数 | `{ev_nd}_cnt_{W}` | `SUM(daily_cnt)` |
| 活跃天数 | `{ev_nd}_days_{W}` | `COUNT(DISTINCT event_date)` |
| 是否发生过 | `{ev_nd}_ever_{W}` | `SIGN(SUM(daily_cnt))` |
| 单日最大次数 | `{ev_nd}_max_daily_{W}` | `MAX(daily_cnt)` |

#### 维度二：日级统计量（avg / std / cv / min / max）

**设计思路**

```
在窗口内，先将事件聚合到天级别（每天发生多少次），
再对这个时间序列做统计。

用户 U 近 28 天的 b1pv 日级序列示例：
  [0, 0, 2, 1, 0, 3, 0, 1, ...]
  → avg_daily_28d = 1.3（日均）
  → std_daily_28d = 1.1（波动）
  → cv_daily_28d  = 0.85（CV = std/avg，低=规律，高=集中爆发）
  → min_daily_28d = 0（含0天，或排除0天取最小正值）
  → max_daily_28d = 3（单天峰值）
```

**模板定义**

| 特征 | 命名规则 | 含义 |
|-----|---------|------|
| 日均次数 | `{ev_nd}_avg_daily_{W}` | 窗口内日均发生次数 |
| 日级标准差 | `{ev_nd}_std_daily_{W}` | 行为波动性 |
| 变异系数 | `{ev_nd}_cv_daily_{W}` | std/avg，行为规律性 |
| 日级最小值 | `{ev_nd}_min_daily_{W}` | 有事件天中最少几次 |
| 单日最大值 | `{ev_nd}_max_daily_{W}` | 单天峰值（与维度一共用） |

#### 维度三：跨窗口比率（Ratio）

**设计思路**

```
ratio = cnt_{短窗口} / cnt_{长窗口}

比率含义：
  ratio → 1.0：行为高度集中在近期（急速上升信号）
  ratio → 0.0：近期无行为，历史行为已冷却
  ratio ≈ 短/长（均匀期望值）：行为均匀分布
```

**6 个标准比率对**

| 比率对 | 命名规则 | 均匀期望值 | 业务含义 |
|-------|---------|---------|---------|
| cnt_7d / cnt_14d | `{ev_nd}_ratio_7_14d` | 0.50 | 近一周加速/减速 |
| cnt_14d / cnt_28d | `{E}_ratio_14_28d` | 0.50 | 近两周趋势 |
| cnt_28d / cnt_56d | `{E}_ratio_28_56d` | 0.50 | 近月趋势 |
| cnt_56d / cnt_112d | `{E}_ratio_56_112d` | 0.50 | 中期趋势 |
| cnt_112d / cnt_224d | `{E}_ratio_112_224d` | 0.50 | 长期趋势 |
| cnt_7d / cnt_all | `{E}_ratio_7_all` | 动态 | 近期行为占历史比重 |

**特征清单示例（以 `loanb1pageview` / `postrepaymentresult` 为例）**

#### 维度四：环比变化（delta / chg_rate / accel）

**设计思路**

```
将时间轴切成等长片段，对比相邻两段的变化，刻画"行为在加速还是减速"：
  近7d vs 前7d   （T-7~T-1  vs  T-14~T-8）
  近14d vs 前14d  （T-14~T-1 vs  T-28~T-15）
  近28d vs 前28d  （T-28~T-1 vs  T-56~T-29）
  近56d vs 前56d  （T-56~T-1 vs  T-112~T-57）
```

**衍生特征类型**

| 特征 | 命名规则 | 计算公式 | 含义 |
|-----|---------|---------|------|
| 绝对变化量 | `{ev_nd}_delta_{W}` | cnt_recent - cnt_prev | 正=增加，负=减少 |
| 相对变化率 | `{ev_nd}_chg_rate_{W}` | (cnt_recent - cnt_prev) / (cnt_prev+1) | 环比增长率（+1防除零） |
| 加速标识 | `{ev_nd}_accel_{W}` | SIGN(cnt_recent - cnt_prev) | +1/0/-1 = 加速/持平/减速 |

**特征清单示例（以 `loanb1pageview` 为例）**

```

// 粒度2：近14d vs 前14d
loanb1pageview_delta_14d       loanb1pageview_chg_rate_14d       loanb1pageview_accel_14d

// 粒度3：近28d vs 前28d
loanb1pageview_delta_28d       loanb1pageview_chg_rate_28d       loanb1pageview_accel_28d

// 粒度4：近56d vs 前56d

```

#### 维度五：时序趋势特征

**设计思路**

```
将用户 × 事件 × 天的数据视为时间序列，提取趋势信息。

示例：用户 U 的 b1pv 日级序列（近56天）：
  [0,0,1,0,0,0,1,0,0,0,2,0,...,3,2,1,3]
                 ↑ 早期偶发       ↑ 近期密集

趋势信号：
  days_since_last  → 距最近一次发生的天数（越小越活跃）
  max_streak       → 连续活跃最长天数（行为持续性）
  max_gap          → 连续不活跃最长天数（失活程度）
  slope            → 线性回归斜率（正=上升趋势）
  ewm              → 指数加权均值（近期偏重的行为水平）
  active_gap_avg/std → 相邻活跃天平均间隔（周期性规律性）
```

#### 维度六：时间间隔特征（intv_min / avg / max / std / cv）

**设计思路**

```

例3：rr（还款结果）连续两次间隔
```

#### 维度七：多事件序列模式（A→B 转化率）

**设计思路**

```
捕捉"用户做了 A 之后多快会做 B"以及"A→B 的完成率"。

核心原理：A 发生 → 在接下来 N 小时/天内 → B 是否发生

对风控模型的价值：
  - KTP准备（A）→ 活体识别（B）的完成率：刻画认证意愿
  - 贷款主页（A）→ 点击申请按钮（B）的转化率：刻画借款决心
  - 还款点击（A）→ 还款成功（B）的成功率：刻画还款能力
```

#### 跨事件组合比率特征

**设计思路**

```
不同事件之间的比率揭示行为结构，而非单一事件频次。

示例：
  authb553d554click_cnt_all / NULLIF(authb553pageview_cnt_all, 0)  → 挽留净成功率
  authb606pageview_cnt_all / NULLIF(authb573pageview_cnt_all, 0)  → 全程认证完成率（KTP→绑卡）
  loanb1d709click_cnt_28d / NULLIF(loanb1pageview_cnt_28d, 0)    → 贷款主页申请转化率
  postrepaymentresult_cnt_112d / NULLIF(postrepaymentresult_cnt_all, 0)          → 近期还款集中度
```

### 模块 B：历史批量回溯（训练/OOT 离线用）

> **与模块 A 完全独立**，不依赖 T+1 人群 base 表，直接从四场景样本表取 `loan_account_id + anchor_date`，可一次性批量计算全部历史样本的特征。

```
            gen_hive_sql.py --mode hist 生成 CASE WHEN 宽表 SQL
```

#### 5.B2 Step1：历史回溯 base 表（四场景合并）

各场景 `anchor_date` 取法：

| 场景 | anchor_date 来源 | 说明 |
|------|-----------------|------|
| 首贷一次风控 | `from_unixtime(trace_created_ts/1000-3600,'yyyy-MM-dd')` | 进入一次风控时刻 |
| 首贷二次风控 | `order_payout_date` 或二次风控进入时刻 | 视建模 anchor 定义 |
| 复贷一次风控 | `from_unixtime(trace_created_ts/1000-3600,'yyyy-MM-dd')` | 同首贷一次 |
| 复贷二次风控 | 同上 | 同首贷一次 |

```sql
CREATE TABLE yanzhang1.yz_event3_hist_base STORED AS PARQUET AS
SELECT DISTINCT user_id, loan_account_id, anchor_date, scene
FROM (
    -- 首贷一次风控
    SELECT user_id, loan_account_id,
           from_unixtime(bigint(trace_created_ts/1000)-3600,'yyyy-MM-dd') AS anchor_date,
           '首贷一次' AS scene
    FROM dm_id.dm_id_risk_df_underwrite
    WHERE trace_risk_type_name IN ('首贷风控','首贷回捞')
      AND dt >= '<历史起始分区>'
    UNION ALL
    -- 首贷二次风控（以 order_payout_date 为锚点示例）
    SELECT user_id, loan_account_id,
           order_payout_date AS anchor_date,
           '首贷二次' AS scene
    FROM dm_id.dm_id_risk_df_underwrite
    WHERE trace_risk_type_name IN ('首贷风控','首贷回捞')
      AND ordertrace_trace_id IS NOT NULL
      AND dt >= '<历史起始分区>'
    UNION ALL
    -- 复贷一次风控
    SELECT user_id, loan_account_id,
           from_unixtime(bigint(trace_created_ts/1000)-3600,'yyyy-MM-dd') AS anchor_date,
           '复贷一次' AS scene
    FROM dm_id.dm_id_risk_df_underwrite
    WHERE trace_risk_type_name IN (
        '首贷还款续借','复贷还款续借','复贷打款续借','结清复贷','循环贷')
      AND dt >= '<历史起始分区>'
    UNION ALL
    -- 复贷二次风控（同复贷一次，anchor_date 按需调整）
    SELECT user_id, loan_account_id,
           from_unixtime(bigint(trace_created_ts/1000)-3600,'yyyy-MM-dd') AS anchor_date,
           '复贷二次' AS scene
    FROM dm_id.dm_id_risk_df_underwrite
    WHERE trace_risk_type_name IN (
        '首贷还款续借','复贷还款续借','复贷打款续借','结清复贷','循环贷')
      AND ordertrace_trace_id IS NOT NULL
      AND dt >= '<历史起始分区>'
) t;
```

两个模块复用，通过 `--mode` 参数切换：

## 七、SQL 实现模板

### 7.1 基础聚合（G1/G2/G5，维度一/二/五）

```sql
DROP TABLE IF EXISTS tmp_export.yz_event3_feature_base;
CREATE TABLE tmp_export.yz_event3_feature_base AS
SELECT
    b.loan_account_id, b.anchor_date, b.mob1_dpd7_ind,

    -- G1: 基础计数（以 auth_b553_pageview 为示例，其余事件同理）
    SUM(CASE WHEN event_code='auth_b553_pageview' AND days_before_anchor<=7  THEN daily_cnt ELSE 0 END) AS authb553pageview_cnt_7d,
    SUM(CASE WHEN event_code='auth_b553_pageview' AND days_before_anchor<=30 THEN daily_cnt ELSE 0 END) AS authb553pageview_cnt_30d,
    SUM(CASE WHEN event_code='auth_b553_pageview'                            THEN daily_cnt ELSE 0 END) AS authb553pageview_cnt_90d,
    COUNT(DISTINCT CASE WHEN event_code='auth_b553_pageview' AND days_before_anchor<=30 THEN event_date END) AS authb553pageview_days_30d,
    SIGN(SUM(CASE WHEN event_code='auth_b553_pageview' AND days_before_anchor<=30 THEN daily_cnt ELSE 0 END)) AS authb553pageview_ever_30d,

    -- G2: 日级统计
    AVG(CASE WHEN event_code='login_b70_pageview' AND days_before_anchor<=7 THEN daily_cnt END) AS loginb70pageview_avg_daily_7d,
    STDDEV(CASE WHEN event_code='login_b70_pageview' AND days_before_anchor<=7 THEN daily_cnt END) AS loginb70pageview_std_daily_7d,

    -- G5: 时序距离（核心特征）
    MIN(CASE WHEN event_code='auth_b573_pageview' THEN days_before_anchor END) AS authb573pageview_days_since_last,
    MIN(CASE WHEN event_code='auth_b553_pageview' THEN days_before_anchor END) AS authb553pageview_days_since_last,
    MIN(CASE WHEN event_code='login_login_result' THEN days_before_anchor END) AS loginloginresult_days_since_last,
    MIN(CASE WHEN event_code='AppStart'            THEN days_before_anchor END) AS AppStart_days_since_last
    -- ... 其余 31 个事件按同等模式扩展 ...

FROM tmp_export.yz_event3_payout_first_base_lite b
LEFT JOIN tmp_export.yz_event3_daily_full d
    ON b.loan_account_id = d.loan_account_id AND b.anchor_date = d.anchor_date
WHERE b.mob1_dpd7_matured = 1
GROUP BY b.loan_account_id, b.anchor_date, b.mob1_dpd7_ind;
```

### 7.2 衍生特征（G3/G4/CV + 跨事件比率）

```sql
CREATE TABLE tmp_export.yz_event3_feature_derived AS
SELECT *,
    -- G3: 跨窗口比率
    authb553pageview_cnt_7d  / NULLIF(authb553pageview_cnt_90d, 0) AS authb553pageview_ratio_7_90d,
    loginb70pageview_cnt_7d  / NULLIF(loginb70pageview_cnt_30d, 0) AS loginb70pageview_ratio_7_30d,
    loginb70pageview_cnt_30d / NULLIF(loginb70pageview_cnt_90d, 0) AS loginb70pageview_ratio_30_90d,

    -- G2: 变异系数 CV
    loginb70pageview_std_daily_7d / NULLIF(loginb70pageview_avg_daily_7d, 0) AS loginb70pageview_cv_daily_7d,

    -- G4: 环比变化（近7d vs 前7d）
    loginb70pageview_cnt_7d - (loginb70pageview_cnt_14d - loginb70pageview_cnt_7d)
        AS loginb70pageview_delta_7d,

    -- 跨事件：认证漏斗质量
    authb553d554click_cnt_90d / NULLIF(authb553d554click_cnt_90d + authb553d555click_cnt_90d, 0)
        AS auth_retain_net_rate_90d,
    authb597pageview_cnt_90d / NULLIF(authb573pageview_cnt_90d, 0) AS lv_conversion_90d,
    authb606pageview_cnt_90d / NULLIF(authb573pageview_cnt_90d, 0) AS bind_conversion_90d,
```

### 7.3 时间间隔特征（G6，需时间戳）

```sql
CREATE TABLE tmp_export.yz_event3_interval_features AS
WITH ranked AS (
    SELECT a.loan_account_id, a.event_code, a.created_time,
        LAG(a.created_time) OVER (
            PARTITION BY a.loan_account_id, a.event_code ORDER BY a.created_time
        ) AS prev_time
    FROM ec_dwd.dwd_ec_log_di_sa_app_events a
    INNER JOIN tmp_export.yz_event3_payout_first_base_lite b
        ON a.loan_account_id = b.loan_account_id
        AND TO_DATE(a.created_time) < b.anchor_date
        AND TO_DATE(a.created_time) >= b.start_90d
    WHERE a.event_code IN ('auth_b594_pageview','auth_b573_pageview','login_b70_pageview','AppStart')
),
intervals AS (
    SELECT loan_account_id, event_code,
        (UNIX_TIMESTAMP(created_time) - UNIX_TIMESTAMP(prev_time)) / 3600.0 AS gap_hours
    FROM ranked WHERE prev_time IS NOT NULL
)
SELECT loan_account_id,
    MIN(CASE WHEN event_code='auth_b594_pageview' THEN gap_hours END) AS auth_b594_intv_min_90d,
    AVG(CASE WHEN event_code='auth_b594_pageview' THEN gap_hours END) AS auth_b594_intv_avg_90d,
    STDDEV(CASE WHEN event_code='auth_b594_pageview' THEN gap_hours END)
        / NULLIF(AVG(CASE WHEN event_code='auth_b594_pageview' THEN gap_hours END), 0)
        AS auth_b594_intv_cv_90d,
    AVG(CASE WHEN event_code='login_b70_pageview' THEN gap_hours END) AS login_b70_intv_avg_90d,
    STDDEV(CASE WHEN event_code='login_b70_pageview' THEN gap_hours END)
        / NULLIF(AVG(CASE WHEN event_code='login_b70_pageview' THEN gap_hours END), 0)
        AS login_b70_intv_cv_90d
FROM intervals
GROUP BY loan_account_id;
```

### 7.4 事件序列模式（G7，A→B 转化率）

```sql
-- P8: 全程认证流完成率：KTP准备 → 活体识别（48h内）
CREATE TABLE tmp_export.yz_event3_seq_features AS
WITH ktp_events AS (
    SELECT a.loan_account_id, a.created_time AS ktp_time
    FROM ec_dwd.dwd_ec_log_di_sa_app_events a
    INNER JOIN tmp_export.yz_event3_payout_first_base_lite b ON a.loan_account_id = b.loan_account_id
    WHERE a.event_code = 'auth_b573_pageview'
      AND TO_DATE(a.created_time) BETWEEN b.start_90d AND b.anchor_date
),
matched AS (
    SELECT k.loan_account_id, k.ktp_time, MIN(l.created_time) AS first_lv_after
    FROM ktp_events k
    LEFT JOIN ec_dwd.dwd_ec_log_di_sa_app_events l
        ON k.loan_account_id = l.loan_account_id
        AND l.event_code = 'auth_b597_pageview'
        AND l.created_time > k.ktp_time
        AND l.created_time <= k.ktp_time + INTERVAL 48 HOUR
    GROUP BY k.loan_account_id, k.ktp_time
)
SELECT loan_account_id,
    COUNT(ktp_time)                                                       AS ktp_to_lv_ktp_cnt_90d,
    SUM(CASE WHEN first_lv_after IS NOT NULL THEN 1 ELSE 0 END)          AS ktp_to_lv_cnt_90d,
    SUM(CASE WHEN first_lv_after IS NOT NULL THEN 1 ELSE 0 END) * 1.0
        / NULLIF(COUNT(ktp_time), 0)                                      AS ktp_to_lv_rate_90d,
    AVG(CASE WHEN first_lv_after IS NOT NULL
             THEN (UNIX_TIMESTAMP(first_lv_after) - UNIX_TIMESTAMP(ktp_time)) / 3600.0 END)
                                                                          AS ktp_to_lv_time_avg_90d
FROM matched
GROUP BY loan_account_id;
```

## 十、风险提示

### 首贷风险

1. **覆盖率偏低**：51 个事件覆盖率 < 1%（占有数据事件 54%），建议合并为事件组或放弃
2. **稳定性风险**：8 个事件月度 CV > 0.3，入模时需持续监控（见 1.1 节 C-不稳定列表）
3. **时间范围限制**：当前 EDA 基于 2025-06 后数据，早期用户行为模式可能不同
4. **B-可用事件**：`authnextstepresult`（IV=0.0095）和 `AppEnd`（IV=0.0056）可作辅助特征，不作核心特征
5. **C-低IV**：`AppPageLeave`（IV=0.0032）和 `AppViewScreen`（IV=0.0027）信息量过低，建议不入模

### 复贷风险

6. **IV 量级偏高需关注**：`loanb1pageview`（IV@30d=0.086）远高于首贷最高 IV（0.021），建议通过 OOT 验证确认真实预测力
7. **防穿越已修复**：daily 表基于用户全局窗口建表，所有回溯 SQL 已加 `AND d.event_date <= b.end_date` 上界
8. **复贷必须用长窗口**：`cov_ratio_7d` 仅 0.08~0.13，短窗口信息量严重不足，必须使用 90d/180d/360d
9. **还款特征合规性**：`postrepaymentresult` 记录的是历史还款行为（非当前申请），属合法历史特征，无数据泄漏风险
10. **一次/二次风控事件池高度相似**：两者 TOP10 事件重合度 > 90%，可复用同一套特征开发代码，仅 anchor_date 字段不同
