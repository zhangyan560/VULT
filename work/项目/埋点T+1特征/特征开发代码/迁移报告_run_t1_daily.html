6. 行为差异（需关注）

差异 1：daily_cnt 滑窗写入 → 单天写入
原脚本：每次刷新最近 3 天分区（T-2/T-3/T-4），用于捕获迟到事件数据。
迁移后：DataPilot 单任务只允许一个 INSERT，每次只写 event_date = ${p_date_iso} 分区。
影响：迟到 2-3 天到达的事件数据不会被补刷。
缓解：如发现上游数据延迟，手动在 DataPilot 重跑 Task02（选择对应历史日期执行）。
差异 2：中间表从 CTAS 全量覆写 → 分区覆写
原脚本：每次运行 DROP TABLE + CREATE TABLE CTAS，中间表只保留当次结果。
迁移后：中间表（feat_a/b/c/d）改为分区表，历史分区保留，便于回溯排查。
影响：存储空间增加（保留历史分区），但下游读取时均加 WHERE dt = '{dt}' 过滤，无数据混淆风险。
差异 3：features 最终表从非分区 → 分区表
原脚本目标表 yz_event3_t1_features 无分区，每次 CTAS 全量重建。
迁移后：按 dt 分区，INSERT OVERWRITE 只覆盖当天分区，历史保留。
优点：支持离线训练时精确回溯任意历史日期的特征分区，防穿越天然保证。
