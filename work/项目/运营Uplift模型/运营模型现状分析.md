当前现状以及分析

当前现状：

运营期望uplift模型用来离线跑批，但运营2026年所开实验没有一个完全干净的跑批实验供模型评估，主要原因是：但线上缺少离线跑批的实验对照组(只有实验13248，其他实验触发模式都不是离线跑批)，而且仅有的离线跑批实验的对照组不干净，会受到其他实验的影响。每个实验的对照组都受到其他实验测试组的干扰。

应对措施：

基于以上现状，我们梳理了以下三个方案：

### 方案1：泳道数据（仅用于特征参考）

样本构成

● 首贷未完件泳道：实验组约50w，对照组约12w

● 复贷泳道：实验组约78w，对照组约8w（截至2026年4月）

优势详解

● 对照组100%不发WA，baseline最干净，适合分析"WA触达"这一干预行为的因果效应

● 样本量大，特征工程（WOE、Target Encoding）的统计估计更稳定

劣势详解

● 泳道内实验组为长期运营实验的混合（涵盖12687/12690/12693/13036等多个不同触达时机与策略的实验），实验组并非单一的"离线跑批WA"行为

● 触达时机的混杂（T0登录触发 vs T1跑批 vs T2还款后触发）意味着学到的是"各种触达混合策略"的效果，而非跑批场景的增量效果

● 结论：方案1适合用于特征有效性筛选（哪些特征与放款率/增量相关），但不适合直接训练最终部署的跑批uplift模型

建议用途

● 结合泳道数据做特征重要性预筛，减少方案2/3训练时的特征搜索空间

● 用泳道对照组（干净基线）做全量用户的放款行为基准分析

### 方案2：复贷泳道跑批实验13248 + T+1观测（❌ 已判定不可用）

实验基本情况

● 实验13248名称：“续借月维度-常规-WA泳道-触达-26-长期实验-3天1次推全-0319重开”

● 实验组：344,129人；对照组：38,140人

为什么不可用（量化结论）

1.  对照组T+1 污染率54.6%：对照组内没有任何一个用户能作为"未被触达"的干净基线。最主要的污染来源是 exp13105（跑批WA，覆盖74.4%）和 exp12912（T2H触达，覆盖71.4%），意味着对照组用户实际上都在被其他实验WA触达，其放款行为混入了其他实验的处理效应。

2.  对照组发Push后 T+1/T+2/T+3 内被其他实验组发WA的时序污染分析

分析口径：仅统计 push_date ≤ 2026-04-06 的用户（确保 T+1/T+2/T+3 三个窗口均有完整数据，避免数据截断导致的可比性问题）。分母为该范围内曾被发过 push 的对照组用户（共 37,252 人）。

|   |   |   |   |
|---|---|---|---|
|T偏移|被其他实验组发WA的用户数|污染率|涉及实验数|
|T+1|20,344|54.6%|80个|
|T+2|20,487|55.0%|83个|
|T+3|20,889|56.1%|86个|

3.  T+1 信号具有代表性（Spearman ρ 均通过），放款率2.5%左右。

4.  实验组/对照组样本严重不均衡：实验组是对照组的9倍（344k vs 38k），即便做权重修正，不均衡本身也会影响 uplift 估计的方差。

结论：T+1 信号本身具有代表性（Spearman ρ 均通过）。但对照组在发Push后的T+1/T+2/T+3内，均有超过54%的用户被其他实验的测试组发了WA——对照组并非"静默基线"，而是持续受到其他实验WA的干扰。污染率在整个观测窗口内始终远超20%阈值，即便缩短至T+1也无法规避。→ 方案2不可用

### 方案3：新开干净实验（正式版训练数据）

实验设计规范

|   |   |
|---|---|
|设计要素|规格|
|实验名称|复贷结清/续借-纯跑批WA-干净实验（2026H2）|
|触达方式|仅离线跑批WA，不含实时触达|
|分流比例|实验组10% + 对照组10%（可后期调整）|
|互斥范围|与 13248、13036、12692、12912 全部互斥|
|观测窗口|T+3（主）、T+7（辅助验证）|
|启动时间|2026年4月下旬（待AB平台配置完成）|
|达量时间|预计2026年7月|

模型训练设计

● 实验组/对照组样本均来自干净实验，理论上 ATE 无偏估计

● 可直接使用 S-learner（简单）/ T-learner（精度更高）/ X-learner（样本不均衡时更稳健）

● Y 变量：T+1 放款标签（0/1），辅以放款金额加权（避免选出小额下单用户）

Part 2：

未来数月运营Uplift模型方向规划

### 整体背景与目标

当前运营策略依赖人工规则节点触达，缺乏对"哪些用户真正会因触达而增量放款"的精准识别。方案2已因对照组持续污染被放弃，方案3（新开干净实验）是唯一可行的正式训练数据来源。本规划以方案3为主线，分阶段将触达决策交给模型，实现精准化、自动化的增量运营。

### 阶段规划总览（2026年4月 ～ 2026年底）

阶段1：得到一个每日离线跑批模型，在当日早上圈选出一波我们认为会有wa增量的用户进行触达，如果用户当日有其他实时行为（登录，还款等）再叠加人工实时触达策略

阶段2: 把实时行为也纳入到模型中，在用户当日用实时行为的时候重新更新标签分，由模型判断是否增加触达，不再做人工规则

阶段3: 把发什么（素材、内容）也一起加入到模型中，不仅让模型选人，也让模型选素材

阶段4: 后期模型迭代、更新、优化如何进行
