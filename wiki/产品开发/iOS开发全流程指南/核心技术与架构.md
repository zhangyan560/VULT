## SwiftUI vs UIKit 核心对比

| 维度 | SwiftUI | UIKit |
|------|---------|-------|
| 推出时间 | 2019年（iOS 13+） | 2008年（随 iPhone OS 1.0） |
| 编程范式 | **声明式（Declarative）** | **命令式（Imperative）** |
| 代码量 | 少（约减少 50%） | 多 |
| 学习曲线 | 较平缓 | 较陡 |
| 实时预览 | Canvas 实时预览 | 需编译后查看 |
| 多平台支持 | iOS/iPadOS/macOS/watchOS/visionOS | 主要 iOS |
| 职场需求 | 新项目主流 | 大量遗留项目仍在使用 |
| **推荐场景** | **独立开发/新项目** | **大厂就业/维护遗留代码** |

> [!tip] 新手建议从 SwiftUI 开始，若目标是大厂就业则两者都需掌握。

## MVVM 架构模式

```
iOS App 推荐架构：MVVM（Model-View-ViewModel）

┌─────────────────────────────────────┐
│           View (SwiftUI)            │  ← 只负责显示，不含业务逻辑
│  @State @Binding @StateObject       │
└────────────────┬────────────────────┘
                 │ 数据绑定
┌────────────────▼────────────────────┐
│         ViewModel (@Observable)     │  ← 业务逻辑层
│   处理用户输入 → 调用 Model → 更新UI │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│              Model                  │  ← 数据结构 + 持久化
│   struct/class + SwiftData          │
└─────────────────────────────────────┘
```

### 关键属性包装器

| 属性包装器 | 作用 | 使用场景 |
|-----------|------|---------|
| `@State` | 本地 View 状态 | 单个 View 内的简单状态 |
| `@Binding` | 父子 View 状态共享 | 子 View 需修改父 View 状态 |
| `@StateObject` | 持有 ViewModel 实例 | 需要整个生命周期持有的对象 |
| `@ObservedObject` | 观察外部 ViewModel | 从父 View 接收 ViewModel |
| `@EnvironmentObject` | 全局共享状态 | App 范围的用户设置等 |
| `@Observable`（iOS 17+）| 新一代状态管理 | 替代 ObservableObject，更简洁 |

## 数据持久化方案对比

| 方案 | 适用场景 | 复杂度 | iOS版本 |
|------|---------|--------|---------|
| **UserDefaults** | 简单键值对（设置项、小数据） | ⭐ | iOS 2+ |
| **SwiftData** | 结构化数据、复杂关系（新项目推荐） | ⭐⭐ | iOS 17+ |
| **CoreData** | 复杂数据库需求、遗留项目 | ⭐⭐⭐ | iOS 3+ |
| **Keychain** | 敏感信息（密码、Token） | ⭐⭐ | iOS 2+ |
| **FileManager** | 文件/图片存储 | ⭐⭐ | iOS 2+ |

> [!note] 新项目推荐优先使用 SwiftData（CoreData 的现代替代方案），学习成本更低且 API 更友好。

## 网络请求 - async/await 模式

```swift
// 现代 iOS 网络请求（Swift 5.5+ async/await）
func fetchData() async throws -> [Item] {
    let url = URL(string: "https://api.example.com/items")!

    // 异步请求，不阻塞主线程
    let (data, _) = try await URLSession.shared.data(from: url)

    // JSON 解码
    return try JSONDecoder().decode([Item].self, from: data)
}

// 在 View 中调用
.task {
    items = try? await fetchData()
}
```

### 网络请求流程

```
用户操作 → View 触发 → ViewModel.fetchData()
    ↓ async/await
URLSession.shared.data(from: url)
    ↓ 解码
JSONDecoder().decode(Model.self, from: data)
    ↓ @MainActor
更新 @Published/@Observable 属性 → View 自动刷新
```

## 推送通知（APNs）

```
推送通知流程：
App → 请求用户授权
    ↓ 用户同意
APNs → 注册 → 获取 Device Token
    ↓
将 Token 发送到后端服务器
    ↓ 服务器触发
APNs → 向目标设备推送通知
```

关键代码：
```swift
// 请求推送权限
UNUserNotificationCenter.current().requestAuthorization(
    options: [.alert, .badge, .sound]
) { granted, error in }
```

## 支付集成（StoreKit 2）

```swift
// iOS 15+ StoreKit 2 现代 API
let products = try await Product.products(for: productIDs)
let result = try await product.purchase()

switch result {
case .success(let verification):
    // 解锁功能
case .userCancelled, .pending:
    break
}
```

> [!important] StoreKit 2（iOS 15+）大幅简化了内购流程，推荐新项目使用。旧项目使用 SKPaymentQueue API。
