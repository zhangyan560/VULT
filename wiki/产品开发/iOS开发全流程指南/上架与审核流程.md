## 完整上架流程总览

```
准备阶段 → Archive归档 → TestFlight测试 → App Store Connect配置 → 提交审核 → 上线
```

## 第一步：证书与配置文件

```
developer.apple.com → Certificates, Identifiers & Profiles

① Identifiers（标识符）
   - 创建 App ID / Bundle ID
   - 格式：com.公司名.应用名（唯一，不可更改）
   - 启用所需 Capabilities（推送、内购、iCloud 等）

② Certificates（证书）
   - Development 证书：用于调试
   - Distribution 证书：用于上架
   - 上传 Mac 生成的 CSR 文件 → 下载并安装到 Keychain

③ Provisioning Profiles（描述文件）
   - 关联：证书 + App ID + 设备列表
   - App Store Distribution：用于正式发布
```

> [!tip] 在 Xcode 中勾选"Automatically manage signing"，Xcode 会自动创建和管理证书和描述文件。

## 第二步：Xcode Archive 与上传

```
Xcode 操作步骤：

1. 顶部 Scheme 选择 "Release" 配置
2. Device 选择 "Any iOS Device (arm64)"（不能选模拟器）
3. 菜单 Product → Archive
4. 弹出 Organizer 窗口 → 选择最新归档
5. 点击 "Distribute App"
6. 选择 "App Store Connect"
7. 选择 "Upload"
8. Xcode 自动验证并上传（约5-10分钟）
```

## 第三步：App Store Connect 元数据配置

登录 appstoreconnect.apple.com，进入 App 记录填写：

### 基本信息
| 字段 | 说明 | 注意事项 |
|------|------|---------|
| 名称 | App 显示名称（≤30字符） | 商店唯一，严禁堆砌关键词 |
| 副标题 | 补充描述（≤30字符） | 不可使用误导性描述 |
| SKU | 内部标识符 | 自定义，用于财务追踪 |
| Bundle ID | 与 Xcode 一致 | 注册后不可更改 |

### 截图规格
| 尺寸 | 对应设备 | 备注 |
|------|---------|------|
| 6.9英寸 | iPhone 16 Pro Max | **必须上传（覆盖所有iPhone）** |
| 12.9英寸 | iPad Pro | iPad 版本需要 |

> [!tip] 在 Xcode 模拟器截图即可。建议用 Canva 或 Figma 制作带文字说明的宣传图。

### ASO 关键词优化
- 100字符的关键词字段（用英文或中文逗号分隔）
- **不要包含 App 名称**（已计入搜索）
- 选择**高热度、低竞争**的词：用 AppFollow / Sensor Tower 分析
- 关键词不显示给用户，但影响搜索排名

### 合规性清单
- [ ] 加密合规（Export Compliance）：无特殊加密算法选"否"
- [ ] 隐私协议 URL（必须，可用 GitHub Pages 托管）
- [ ] 支持页面 URL
- [ ] 版权信息（如：© 2025 Your Name）
- [ ] 年龄分级（根据内容如实填写）
- [ ] 隐私标签（Privacy Labels）：说明数据收集范围

## 第四步：TestFlight Beta 测试

```
内部测试（Internal Testing）：
- 最多 100 人
- 无需苹果审核，上传后立即可用
- 适合：开发团队、密友测试

外部测试（External Testing）：
- 最多 10,000 人
- 需要苹果轻量级审核（通常 24 小时内）
- 可通过公开链接分发（发布到 Discord/社媒/邮件）
- 每个构建版本有效期 90 天
```

### 分发方式
1. 邮件邀请：输入测试者 Apple ID
2. 公开链接：生成链接任何人可加入
3. Redemption Code：生成兑换码

## 第五步：提交审核

1. 在 App Store Connect 中选择上传的构建版本
2. 完成所有元数据填写
3. 点击"Add for Review" → "Submit to App Review"
4. 等待 **24-48 小时** 收到审核结果

## 常见被拒原因与解决方案

| 被拒原因 | 解决方案 |
|---------|---------|
| 元数据堆砌关键词/误导性描述 | 修改副标题，使用简洁准确描述 |
| 未提供测试账号 | 在备注中填写演示账号密码 |
| 内购未通过 Sandbox 测试 | 使用 Sandbox 测试账号完整走通内购流程 |
| 加密合规未填写 | Export Compliance 勾选"No" |
| 崩溃/严重 Bug | 修复后重新 Archive 上传 |
| 功能描述与实际不符 | 确保截图和描述与 App 实际功能一致 |
| 隐私标签不准确 | 如实填写所有数据收集情况 |
| App 与系统功能重复过多 | 增加独特功能点，确保有差异化价值 |

> [!note] 被拒后不要慌，苹果会提供详细修改建议。修改后可以"Reply to Reviewer"重新申诉或直接重新提交。

## 审核后发布选项

| 发布方式 | 说明 |
|---------|------|
| 自动发布 | 审核通过后立即上架（推荐） |
| 手动发布 | 审核通过后，由开发者手动触发上架（适合配合营销活动） |
| 定时发布 | 设定特定时间自动上架 |
| 分阶段发布（Phased Release）| 7天内逐步推送给用户（适合大版本更新，降低风险） |
