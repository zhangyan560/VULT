## 学习时间曲线

```
Phase 1：基础学习期（0 → 第一个可运行的 App）
时间：4-8 周（每天2-3小时学习）

内容：
- Swift 语言基础（1-2周）
- SwiftUI View 构建（1-2周）
- 数据管理 @State/@Binding（1周）
- 完成第一个简单 App（1-2周）

里程碑：能独立构建5个界面以内的简单 App

────────────────────────────────────────

Phase 2：MVP 开发期（第一个 App → 可上架）
时间：1-3 个月（全职约1个月，兼职约2-3个月）

内容：
- 网络请求 + API 集成
- 数据持久化（SwiftData）
- 用户认证（Sign in with Apple / Firebase）
- 推送通知
- 内购/订阅（StoreKit 2）
- UI 打磨 + 适配不同设备

里程碑：完整功能 MVP，可提交 TestFlight

────────────────────────────────────────

Phase 3：上架审核期
时间：1-2 周

内容：
- App Store Connect 配置（元数据、截图）
- 证书与签名配置
- TestFlight Beta 测试（1周）
- 提交审核（24-48小时审核周期）
- 根据被拒反馈修改并重新提交

里程碑：App 成功上架 App Store

────────────────────────────────────────

Phase 4：迭代维护期（上架后持续）
时间：持续进行

内容：
- 用户反馈修复 Bug
- iOS 新版本适配（苹果每年9月发布新iOS）
- 功能迭代
- ASO 优化（关键词、截图 A/B 测试）
- 营销与增长

里程碑：实现 MRR 可持续增长
```

## 从零到上架的现实时间估算

| 场景 | 时间 | 前提条件 |
|------|------|---------|
| 纯新手，兼职学习 | 6-12 个月 | 每天1-2小时 |
| 有编程基础，兼职 | 3-6 个月 | 有其他语言经验 |
| 有 Web 开发经验，全职 | 1-3 个月 | 每天8小时 |
| 有 iOS 经验，构建简单 App | 2-4 周 | — |

> [!note] "How to Make an App in 8 Days"（CodeWithChris）展示了有基础的开发者可以在8天内完成一个功能完整的 App，但不包括上架审核时间。

## 常见技术坑点

| 坑点 | 描述 | 解决方案 |
|------|------|---------|
| Xcode 版本问题 | 系统要求 macOS 版本与 Xcode 不匹配 | 及时更新 macOS 和 Xcode |
| 证书过期 | 开发证书有效期1年，到期 App 无法编译 | 设置提醒，提前续签 |
| TestFlight 构建过期 | 每个构建 90 天后自动失效 | 定期上传新构建 |
| SwiftUI 版本兼容 | iOS 16 以下部分 API 不可用 | 设置 Deployment Target，使用 `#available` 判断 |
| 内购沙盒测试 | Sandbox 与 Production 环境差异 | 严格在真机上使用 Sandbox 账号测试 |
| App 被拒循环 | 多次被拒同一原因 | 仔细阅读拒审邮件，善用 Resolution Center 申诉 |
| 审核等待焦虑 | 第一次提交往往最慢（3-7天） | 做好等待准备，提前准备好修改方案 |

## AI 工具加速开发趋势（2025）

> 来源视频显示，2024-2025 年独立开发者正在大量使用 AI 工具加速开发：
- **GitHub Copilot / Claude / Cursor**：生成样板代码、解决 Bug
- **目标**：将 MVP 开发周期压缩到 10 天内
- **适合场景**：工具类 App、简单社交功能、内容消费类 App
