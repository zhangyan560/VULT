# 开发框架与技术选型

![[assets/infographic-开发框架.png|600]]

## 三大框架核心对比

| 维度 | 原生开发 (Swift/Kotlin) | Flutter | React Native |
| :--- | :--- | :--- | :--- |
| **性能表现** | 顶级：最流畅的"奶油般"体验 | 极高：编译为机器码，Skia 绘图引擎 | 一般：JavaScript 桥接原生组件 |
| **开发效率** | 较低：维护 iOS + Android 两套代码 | 高：一套代码跨平台，支持热重载 | 高：一套代码跨平台，热重载 |
| **学习曲线** | 陡峭：需掌握 Swift + Kotlin | 中等：需学 Dart 语言 | 平缓：基于 JavaScript/React |
| **社区生态** | 最强：所有第三方库均有原生支持 | 成长中：Google 背书，Firebase 集成极佳 | 非常强：全球最大 UI 社区之一 |
| **适用场景** | 性能极限游戏、复杂动画、单平台 | 高定制化 UI、Google 生态项目 | 快速迭代商业应用、Web 技术栈团队 |

## 针对不同开发者的选择建议

### 独立开发者 → React Native (Expo)

核心目标是快速验证市场并盈利。Expo 将流程简化到极致：一套代码同时发布两个平台，遇到 Bug 时 JavaScript 资料最多。

> [!tip] 推荐技术栈
> 移动端：React Native + Expo
> 后端：Supabase 或 Firebase
> 网页端（如需要）：Next.js

### 小团队 / 初创 → Flutter 或 React Native

- 追求高定制化 UI → **Flutter**
- 追求招聘容易度和生态成熟度 → **React Native**

### 企业 → 原生开发 或 React Native

大厂（Discord、Tesla、Uber）均采用 React Native 来平衡开发速度与规模。

## 决策树

```
是否需要极致性能和复杂底层动画？
├── 是 → 原生开发 (Swift/Kotlin)
└── 否 ↓

是否要同时覆盖 iOS + Android？
├── 否（单平台）→ 原生开发
└── 是 ↓

团队是否已熟悉 JavaScript / React？
├── 是 → React Native
└── 否 ↓

是否需要高度定制 UI 且依赖 Google 服务？
├── 是 → Flutter
└── 否（需要大量第三方 SDK）→ React Native
```

## MVP 阶段推荐技术栈

| 层级 | 推荐工具 | 原因 |
| :--- | :--- | :--- |
| 移动端 | React Native + Expo | 开发支持最广泛，冷启动最快 |
| 后端 | Supabase / Firebase | 无需运维，直接接入数据库 |
| AI 功能 | OpenAI API | 按需付费，无需训练模型 |
| 快速原型 | Emergent / Cursor | AI 辅助编码，数天内完成 MVP |

## 关键原则

> [!important] 核心决策逻辑
> - 追求**极致体验** → 原生
> - 追求**开发速度** → React Native
> - 追求**高性能 + 定制 UI** → Flutter
> - **初创项目：先上线、后优化**，首选跨平台框架
