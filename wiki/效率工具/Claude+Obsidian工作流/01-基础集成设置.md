# Claude+Obsidian 基础集成设置

> [!tip] 核心定位
> Obsidian 是"文件柜"（存储层），Claude Code 是"执行层"（大脑），两者通过本地文件系统直接共生——数据完全归你所有，无供应商锁定。

## 架构三层

| 层级 | 工具 | 职责 |
|------|------|------|
| 存储层 | Obsidian Vault | 管理本地 Markdown 文件，维护双向链接 |
| 执行层 | Claude Code CLI | 读写文件、执行命令、搜索网络 |
| 协议层 | MCP | 连接外部工具（日历/Slack/GitHub） |

## 启动方式

```bash
# 在 Vault 文件夹内启动
cd ~/Documents/MyVault
claude

# 自动扫描并生成 claude.md
/init
```

## claude.md 的"前额叶皮层"角色

`claude.md` 是系统的核心配置文件，定义三件事：
- **项目约定**：文件夹结构、命名规范、Frontmatter 模板
- **风格约定**：写作语气、标签体系、链接规则
- **禁止清单**：AI 不应该做的事（如不修改 `.obsidian/` 配置、不删除笔记）

> [!important] 维护纪律
> Claude Code 开发者 Boris 建议：每周多次更新 claude.md 以修正错误路径，将其视为"活文档"而非一次性配置。

## 初始文件夹结构

```
Vault/
├── daily-notes/     # 日记（YYYY-MM-DD.md）
├── projects/        # 项目笔记
├── research/        # 调研
├── inbox/           # 收件箱（未整理想法）
└── assets/          # 图片附件
```

## 核心优势

- **数据所有权**：全部本地 Markdown，Obsidian CEO 明确无 AI 原生路线图，避免供应商锁定
- **双向链接**：Claude 理解 `[[wikilink]]` 关联，能自动建立知识网络
- **零门槛启动**：无需安装额外插件，在 Vault 目录执行 `claude` 即可

## 适用人群

**新手**：Brain Dump 工作流——把碎片想法、录音转录粘进 Claude，让它整理成符合 Obsidian 规范的 Markdown 并自动建链。
