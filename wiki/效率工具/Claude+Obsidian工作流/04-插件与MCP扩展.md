# Claude+Obsidian 插件与MCP扩展

> [!tip] 核心定位
> MCP（模型上下文协议）是扩展 Claude 感知和行动能力的"神经连接层"，让 AI 从只能读写本地文件，进化为能操控整个数字工作流的"幕僚长"。

## Obsidian REST API 插件

**用途**：允许外部工具（Claude Desktop、脚本）通过 API 密钥直接操作 Vault。

**安装步骤**（Zen van Riel 方案）：
1. Obsidian 设置 → 社区插件 → 搜索 "Local REST API"
2. 安装并启用，在插件设置中复制 API Key
3. 在 Claude Desktop 的 MCP 配置中填入本地端口（默认 `27123`）

**支持操作**：创建笔记、搜索全库、读取/修改文件内容

## Tasknotes MCP

**核心差异**：每个任务 = 独立 Markdown 文件（非清单列表）。

**优势**：
- AI 可批量处理：扫描所有任务文件，统一更新状态
- 支持任务排期：自动将任务写入日历
- 便于关联：每个任务文件可链接到相关笔记、会议记录

**使用场景演示（Antone Heyward）**：
```
用户："把所有 overdue 的任务重新排期到下周"
Claude 扫描 tasks/ 文件夹 → 识别逾期任务 → 批量修改 due_date
```

## Context Mode MCP（Better Stack 方案）

**问题**：大型文件（CSV、日志、知识库导出）直接进上下文 → Token 爆炸。

**原理**：
```
原始大文件
    ↓ 索引
本地 SQLite 数据库
    ↓ 查询（仅返回相关片段）
Claude 上下文窗口
```

**效果**：Token 消耗降低 **99%**，支持建立"会话检查点"。

**安装**：
```bash
# 通过 UV 管理 Python 依赖
uv add context-mode-mcp
# 在 claude.json 注册 MCP 服务器
```

## 三级集成方案对比

| 级别 | 工具组合 | 适合人群 |
|------|---------|---------|
| **基础** | Obsidian REST API + Claude Desktop | 非技术用户，想用自然语言操作笔记 |
| **进阶** | MCP 编排器（Slack/日历/GitHub） | 知识工作者，需要跨平台信息汇总 |
| **高阶** | Context Mode + Python 过滤层 | 开发者，追求极低成本和安全隔离 |

## 零信任配置原则

> [!warning] 安全提示
> 接入外部工具时，从**只读权限**开始，逐步扩展。不要在初始配置时就授予 AI 发送邮件或修改日历的权限。

实施步骤：
1. 先配置只读 MCP（读取日历/邮件）
2. 验证 AI 行为符合预期后，开放草稿权限
3. 敏感操作（发送/删除）始终保留人工审批节点
