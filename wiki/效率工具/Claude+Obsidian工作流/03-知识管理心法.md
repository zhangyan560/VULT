# Claude+Obsidian 知识管理心法

> [!important] 核心矛盾
> AI 能极大提升效率，但也可能污染你的"思维圣殿"。心法的本质是：**让 AI 成为思考伙伴，而非思考替代者**。

## IDI 框架（Nick Milo）

```
想象（Imagine）  →  辨别（Discern）  →  整合（Integrate）
利用AI探索可能性     保持人类判断力       将洞见纳入知识体系
```

## 防御 vs 进攻：双轨策略

### 防御：AI Zone 隔离

建立专门的 AI 文件夹，与个人原创笔记**完全隔离**：

```
Vault/
├── personal/        # 个人原创——禁止 AI 直接修改
│   └── ideas/
└── ai-zone/         # AI 生成内容专区
    ├── summaries/
    ├── drafts/
    └── analysis/
```

**目的**：防止 AI 废话稀释个人声音，保护"思维圣殿（Ideaverse）"的纯净度。

### 进攻：深度思维命令

| 命令 | 功能 |
|------|------|
| `/trace` | 追踪某个想法在库中跨越数月/数年的演变历史 |
| `/drift` | 对比"当初意图"与"实际行为"的偏差 |
| `/emerge` | 从零散前提中提取潜在的、用户未察觉的模式 |

## 知识飞轮效应（Rick Mulready）

```
Brain Dump（碎片输入）
        ↓
   Claude 结构化整理
        ↓
  Obsidian 库积累增长
        ↓
  AI 背景知识越来越丰富
        ↓
   输出质量持续提升
        ↑_____________↓
         （闭环强化）
```

> [!tip] 关键洞察
> 这个飞轮不需要你"训练"模型——你只需要持续向库里输入原始想法，AI 会随着库的增长变得越来越"懂你"。

## 验证驱动原则（AI LABS × Anthropic）

来自 Claude Code 开发者 Boris 的建议：

> 不要只给 AI 下达任务，要让它在执行前说明**如何验证**自己的工作。

实践方式：
- 在 claude.md 中添加规则："执行任何写入操作前，先列出验证方案"
- 验证方式举例：Python 测试脚本、浏览器扩展检查、模拟器预览

## 零信任权限原则

```
阶段 1：只读权限
  Claude 可读取：日历、邮件、Slack（不可发送）
        ↓
阶段 2：草稿权限
  Claude 可生成：邮件草稿、任务列表（需人工确认）
        ↓
阶段 3：受限写入
  Claude 可执行：更新任务状态、归档笔记（特定范围）
```

## "致命三角"安全警告

当 AI 同时拥有以下三项时，存在提示词注入风险：

- **私有数据访问**（邮件/日历/笔记）
- **不可信内容输入**（外部邮件/网页内容）
- **外发数据通道**（发送消息/API 接口）

**对策**：使用 Python API 过滤层，对敏感操作设置人工审批节点。
