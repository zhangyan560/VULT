# Claude+Obsidian 自动化脚本技巧

> [!tip] 核心思路
> 通过 Skills（斜杠命令）+ Hooks（钩子）+ Context Mode（虚拟化），构建一个低 Token 消耗、高自动化的工作飞轮。

## Claude Skills（自定义斜杠命令）

存放位置：`.claude/skills/` 目录下的 Markdown 或 Python 文件。

| 命令 | 功能 |
|------|------|
| `/today` | 整合日历+待办+Slack，生成每日计划 |
| `/research` | 抓取网页、汇总信息，写入 Obsidian |
| `/new` | 按模板创建新笔记并自动填充 Frontmatter |
| `/presentation` | 触发 PowerPoint/Excalidraw 生成逻辑 |

### 渐进式披露（Progressive Disclosure）

**问题**：将所有 MCP 工具描述预加载进上下文会导致 Token 膨胀。

**解决**：仅在调用特定 `/skill` 时，才加载对应的详细指令——未调用的技能不占上下文空间。

```
用户输入 /presentation
  → Claude 加载 .claude/skills/presentation.md
  → 执行 PowerPoint 生成逻辑
  → 完成后指令从上下文移除
```

## Hooks（自动触发钩子）

### Session Start Hook

每次启动 Claude 自动加载记忆文件：

```markdown
# soul.md    — AI 性格定义（语气、风格）
# user.md    — 用户背景（职业、偏好、目标）
# memory.md  — 关键决策记录（跨会话持久化）
```

### Pre-compact Hook

对话上下文即将压缩前，自动将关键见解保存到 Obsidian 每日日志——解决 Claude 的"会话失忆"问题。

## Context Mode 虚拟化

**问题**：大型 CSV、日志文件直接投喂 Claude → Token 消耗爆炸。

**解决**：Context Mode MCP 将大文件索引到本地 **SQLite 数据库**，查询时仅返回相关片段。

> [!important] 效果
> Token 消耗降低 **99%**，同时支持建立"会话检查点"，类似游戏存档。

## Brain Dump 自动整理流程

```
1. 粘贴碎片想法 / 录音转录 / 乱序笔记
       ↓
2. Claude 识别主题，拆分为独立笔记
       ↓
3. 自动添加 Frontmatter（title/date/tags）
       ↓
4. 建立 [[双向链接]]，关联现有笔记
       ↓
5. 按文件夹结构归档
```
