# Claude+Obsidian 全局工作流优化方案

> [!important] 核心约束
> 工作目录不固定——可能在 `~/VULT`、`~/code/项目`、`~/Downloads` 等任意位置。
> 解决方案不能依赖"在 Vault 目录里启动"，必须**全局有效**。

## 架构设计

```
任意工作目录
    ↓ 启动 Claude Code
~/.claude/CLAUDE.md        ← 每次必加载（全局规则 + 记忆引用）
    ├── @RTK.md            ← Token 过滤代理
    ├── @memory/user.md    ← 用户背景（跨目录持久）
    └── @memory/memory.md  ← 关键决策记录（跨会话持久）
    ↓
项目级 claude.md（可选）  ← 项目特定规则，不重复全局规则
```

**Obsidian Vault 访问**：始终用绝对路径 `/Users/zhangyan/VULT/`，无需切换目录。

---

## 实施清单

### 步骤 1：建立全局记忆层

在 `~/.claude/memory/` 创建两个文件：

**`user.md`** — 用户背景（Claude 每次都能"认识你"）：
```markdown
# 用户背景
- Obsidian Vault 路径：/Users/zhangyan/VULT
- 工作语言：简体中文（代码英文）
- 常用技术栈：（填入你的技术栈）
- 工作风格：（填入偏好）
```

**`memory.md`** — 跨会话关键决策记录（Claude 自动维护）：
```markdown
# 关键决策记录
<!-- Claude 在发现重要决策或反复出现的模式时，追加记录 -->
<!-- 格式：[YYYY-MM-DD] 决策内容 → 原因 -->
```

### 步骤 2：更新全局 CLAUDE.md

在 `~/.claude/CLAUDE.md` 开头添加：
```
@memory/user.md
@memory/memory.md
```

这样每个会话，无论在哪个目录，Claude 都会自动加载你的背景和历史决策。

### 步骤 3：创建 Obsidian 写入技能

在 `~/.claude/skills/obsidian-note/` 创建技能，调用 `/obsidian-note` 可从**任意目录**向 Vault 写入笔记：

```markdown
# 技能：创建 Obsidian 笔记

Vault 绝对路径：/Users/zhangyan/VULT

接收参数：
- 标题（必填）
- 分类（填入对应文件夹：daily-notes / projects / research / AI工具 / 商业研究 / 效率工具）
- 内容

执行：
1. 确认目标路径（绝对路径）
2. 生成符合 Vault 规范的 Frontmatter
3. 写入文件（Write tool 使用绝对路径）
4. 输出文件路径供确认
```

### 步骤 4：项目级 claude.md 极简模板

新项目创建时，只需写**项目特定内容**，全局规则不重复：

```markdown
# 项目：[名称]

## 技术栈
[填写]

## 特殊约定
[与全局规则不同的地方才写]

## Obsidian 关联
对应笔记：/Users/zhangyan/VULT/projects/[项目名]/
```

## 核心工作流

### 日常笔记流（任意目录可用）

```
1. 在任意项目目录启动 Claude
2. Claude 自动加载 user.md + memory.md（来自全局 CLAUDE.md）
3. 需要记录到 Obsidian 时：
   "把刚才的结论写入 Obsidian，路径 /Users/zhangyan/VULT/projects/X/..."
   Claude 用绝对路径直接写入——无需切换目录
```

### 知识归档流

```
项目工作中产生洞见
    ↓
"把这个记录到 VULT 的 research/ 下"
    ↓
Claude 写入 /Users/zhangyan/VULT/research/[日期]-[主题].md
    ↓
双向链接手动或由 Claude 补充
```

### 关键决策记录流

```
做了重要技术决策
    ↓
"记录到 memory.md"
    ↓
Claude 追加到 ~/.claude/memory/memory.md
    ↓
下次任意目录启动时自动加载——Claude "记得"这个决策
```

## 与现有配置的整合

| 现有配置 | 角色 | 优化后 |
|---------|------|-------|
| `~/.claude/CLAUDE.md` | 全局规则 | 增加 `@memory/user.md` `@memory/memory.md` 引用 |
| `~/.claude/RTK.md` | Token 过滤 | 不变 |
| `settings.json` hooks | PreToolUse 安全拦截 | 不变（RTK 自动生效） |
| `VULT/CLAUDE.md` | Vault 规范 | 不变（只在 VULT 内生效） |
| 项目级 `claude.md` | 项目约定 | 极简化，只写特殊内容 |

---

## 不推荐的方案

| 方案 | 原因 |
|------|------|
| Obsidian REST API 插件 | 需要 Obsidian 保持运行，且增加安全暴露面 |
| Session Start Hook 加载记忆 | Claude Code 目前没有 SessionStart hook 类型 |
| 在 VULT 目录下处理所有工作 | 违背实际工作习惯，强制切换目录增加摩擦 |
| 把所有规则塞进全局 CLAUDE.md | 超过 200 行导致每次会话 token 消耗飙升 |
