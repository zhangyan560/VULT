# 电商平台数据抓取方案

> Amazon、Shopify 独立站、eBay 等平台的商品信息、评价、销量趋势采集方案。

## 一、电商数据采集核心目标

- **竞品价格监控**：实时跟踪竞争对手定价策略
- **评价挖掘**：批量获取用户评论用于情感分析
- **库存趋势**：监控畅销品库存变化（间接推断销量）
- **关键词研究**：分析产品标题、Bullet Points 的关键词布局

### 2.2 API 服务方案（生产推荐）

**Bright Data**：
- 提供 Amazon 专属 API
- 支持抓取 ASIN 详情、卖家信息、多页翻页
- 内置代理轮换，绕过反爬保护

**SerpApi**：
- 结构化 Amazon 搜索结果 API
- 按次计费，适合中等规模采集

### 2.3 Python 代码方案

```python
import requests
from bs4 import BeautifulSoup

headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    "Accept-Language": "en-US,en;q=0.9",
}

def scrape_amazon_product(asin: str) -> dict:
    url = f"https://www.amazon.com/dp/{asin}"
    resp = requests.get(url, headers=headers)
    soup = BeautifulSoup(resp.content, "html.parser")

    return {
        "title": soup.find("span", id="productTitle").get_text(strip=True),
        "price": soup.find("span", class_="a-price-whole").get_text(strip=True),
        "rating": soup.find("span", class_="a-icon-alt").get_text(strip=True),
    }
```

> [!warning] 注意
> Amazon 反爬严苛，生产环境必须使用住宅代理 + 随机延迟。直接请求极易被封 IP。

## 三、Shopify 独立站采集方案

### 3.1 利用 Shopify 公开 API 端点

所有 Shopify 店铺都暴露公开产品 JSON：
```
https://{store_domain}/products.json?limit=250&page=1
```

可直接获取产品名称、价格、变体、库存状态，无需爬虫。

### 3.2 店铺识别工具

- **Shopify Spy**：浏览器扩展，识别 Shopify 站点并下载产品集合
- **BuiltWith**：检测目标站点是否使用 Shopify

### 3.3 AliExpress → Shopify 数据导入

使用 MyDataProvider 等服务将 AliExpress 商品数据直接导入 Shopify 店铺，适合选品调研。

## 四、eBay 采集方案

- **官方 API**：eBay Finding API / Browse API，免费访问商品列表和价格历史
- **网页解析**：Scrapy + 旋转代理，适合超出 API 限制的大规模采集

---

## 五、技术选型决策树

```
需要采集电商数据
├── 数据量小（<1000条）→ 浏览器插件（Tapicker/ParseHub）
├── 中等规模，预算有限 → Python + BeautifulSoup + 免费代理
├── 大规模，高频更新 → Bright Data / SerpApi（付费服务）
└── 需要历史价格数据 → Keepa API（Amazon专属）
```

---

## 六、反爬注意事项

- Amazon 必须使用住宅代理，数据中心 IP 极易被封
- 设置随机延迟（2-5 秒），模拟真实用户
- 使用 Session 保持 Cookie，避免每次请求重新建立会话
- 监控响应状态码（429 = 被限速，503 = 被封禁）
