## 一、现代反爬虫防护体系

现代反爬体系已从简单的 IP 封禁演变为多维指纹识别系统。

### 1.1 Cloudflare 防护机制

**被动检测（Passive）**：
- **IP 信誉评分**：VPN/数据中心 IP 直接降分
- **HTTP Header 分析**：User-Agent、Accept-Language 是否符合浏览器特征
- **TLS 指纹识别**：TLS 握手序列模式，不同库有不同的"指纹"

**主动挑战（Active）**：
- **JavaScript 挑战**：执行一段 JS 代码，真实浏览器通过，裸 HTTP 请求卡死
- **CAPTCHA**：触发验证码（hCaptcha/reCAPTCHA）
- **行为追踪**：鼠标移动路径、滚动模式、点击时序

## 二、TLS 指纹伪造：curl-cffi

当请求返回 403 时，大概率是 TLS 指纹被识别：

```python
# 安装
# pip install curl-cffi

from curl_cffi import requests

# impersonate 参数让 TLS 握手模拟真实 Chrome
response = requests.get(
    "https://target-site.com/data",
    impersonate="chrome"  # 可选: chrome, firefox, safari, edge
)
print(response.status_code)  # 通常直接返回 200
```

**原理**：`curl-cffi` 在底层使用 curl 并模拟浏览器的 TLS Client Hello，包括 Cipher Suites 顺序和扩展字段。

## 三、代理轮换策略

### 3.1 代理类型选择

| 代理类型 | 价格 | 被封风险 | 适用场景 |
|----------|------|----------|----------|
| 数据中心代理 | 低 | 高 | 低防护站点、开发测试 |
| 住宅代理 | 中 | 低 | Amazon、Instagram 等高敏感平台 |
| 移动代理（4G/5G） | 高 | 极低 | 最高防护等级的平台 |

> [!important] 核心原则
> 采集 Amazon、LinkedIn 等平台**必须**使用住宅代理，数据中心代理几乎必封。

### 3.2 轮换策略

```python
import requests
import random

PROXY_POOL = [
    "http://user:pass@residential-proxy-1:8080",
    "http://user:pass@residential-proxy-2:8080",
    # ... 更多代理
]

def get_with_rotation(url: str, headers: dict) -> requests.Response:
    proxy = random.choice(PROXY_POOL)
    return requests.get(
        url,
        headers=headers,
        proxies={"http": proxy, "https": proxy},
        timeout=15
    )
```

**粘性会话（Sticky Session）**：在同一逻辑流程内保持同一 IP 3-10 分钟，避免 IP 瞬变触发安全警报。

## 四、浏览器自动化方案

### 4.1 Playwright（推荐）

```python
from playwright.async_api import async_playwright
import asyncio

async def scrape_with_playwright(url: str) -> str:
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()

        # 模拟人类行为
        await page.goto(url)
        await page.mouse.move(300, 400)  # 移动鼠标
        await page.evaluate("window.scrollTo(0, 300)")  # 滚动
        await asyncio.sleep(random.uniform(1, 3))  # 随机延迟

        content = await page.content()
        await browser.close()
        return content
```

### 4.2 NoDriver / 浏览器指纹管理

- 使用 `nodriver` 库：默认模拟正常浏览器行为
- 手动登录一次后导出 Session Cookies JSON，后续脚本加载以绕过登录

## 六、合规边界

> [!warning] 法律风险
> 数据采集需遵守目标网站的服务条款（ToS）和相关法律。

**基本原则**：
1. **robots.txt**：始终检查并遵守，`Disallow` 的路径不要爬取
2. **速率限制**：不超过人类正常浏览频率（建议 2-5 秒延迟）
3. **个人信息**：涉及用户个人可识别信息（PII）需要额外的合规处理
4. **ToS 边界**：竞争对手分析通常是灰色地带，但批量抓取通常被明令禁止

**常见合规方案**：
- 只采集公开可访问的数据
- 不绕过身份验证机制
- 数据仅用于内部研究，不对外发布原始数据

## 七、反爬应对 SOP

```
请求返回异常
├── 403 Forbidden → TLS 指纹问题 → 使用 curl-cffi
├── 429 Too Many Requests → 速率限制 → 增加延迟 + 代理轮换
├── 503 Service Unavailable → IP 封禁 → 切换住宅代理
├── CAPTCHA 页面 → 需要浏览器自动化 → Playwright + CAPTCHA 服务
└── 空白/异常 HTML → JS 渲染页面 → 必须用浏览器引擎
```
