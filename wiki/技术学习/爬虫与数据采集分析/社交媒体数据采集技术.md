# 社交媒体数据采集技术

> 覆盖 Reddit、Twitter（X）、Facebook 三大平台的公开帖文、评论及互动数据采集方案。

## 一、技术选型原则：API 优先 vs 网页解析

| 维度 | API 优先 | 网页解析（HTML/XHR） |
|------|----------|----------------------|
| 稳定性 | 高，官方支持 | 低，依赖页面结构 |
| 数据量 | 受速率限制 | 理论无限制 |
| 合规性 | 合规 | 需注意 ToS |
| 维护成本 | 低 | 高（随页面更新变化） |
| 适用场景 | 实时监控、学术研究 | API 限制严苛时的备选 |

## 二、Reddit 采集方案

### 2.1 官方 API + PRAW（推荐首选）

**PRAW**（Python Reddit API Wrapper）是 Reddit 官方推荐的 Python 客户端。

```python
import praw

reddit = praw.Reddit(
    client_id="YOUR_CLIENT_ID",
    client_secret="YOUR_CLIENT_SECRET",
    user_agent="MyScript/1.0"
)

# 抓取 subreddit 热帖
subreddit = reddit.subreddit("Python")
for post in subreddit.hot(limit=100):
    print(post.title, post.score, post.url)
    # 递归获取评论树
    post.comments.replace_more(limit=None)
    for comment in post.comments.list():
        print(comment.body)
```

**API 限制**：免费账号每分钟 60 次请求，历史数据受限。

### 2.2 存档工具：Pushshift

对于历史数据（超出 API 时间窗口）：
- Pushshift 允许搜索归档的帖文和评论，突破部分 API 限制
- 适用于学术研究、大规模历史语料构建

### 2.3 简易 JSON 方法

Reddit 支持在 URL 末尾添加 `.json` 获取原始数据：
```
https://www.reddit.com/r/Python/hot.json?limit=25
```

## 三、Twitter（X）采集方案

### 3.1 官方 API v2

- **Basic Tier**：每月 150 万 tweet 读取，500k tweet 写入
- **Free Tier**：已大幅限制，仅供发布自己的推文
- **适合**：需要大量数据的企业/研究机构

### 3.2 XHR 请求截获（无 API 方案）

通过浏览器开发者工具截获网络请求：
1. 打开 Chrome DevTools → Network → XHR
2. 在 Twitter 上执行搜索操作
3. 找到包含 JSON 数据的请求，复制为 Python 代码
4. 转化为 Requests 脚本，设置同等 Headers 和 Cookies

**注意**：此方案依赖浏览器会话状态，不稳定。

### 3.3 公开数据集

对 NLP 训练：直接使用 **Sentiment140**（160 万条标注推文），避免从零采集。

## 四、Facebook 采集方案

Facebook 的反爬能力最强，建议：
- **Graph API**：需要应用审核，适合公开主页数据
- **公开主页 HTML 抓取**：使用 Playwright 模拟浏览器，但风险最高
- **第三方 API 服务**：Apify、Smartproxy 提供托管抓取服务

---

## 五、工具生态对比

| 工具 | 类型 | 适用平台 | 优势 |
|------|------|----------|------|
| PRAW | Python 库 | Reddit | 官方支持，功能完整 |
| Scrapy | 爬虫框架 | 通用 | 高性能，分布式支持 |
| Playwright | 浏览器自动化 | 所有 | 处理 JS 渲染页面 |
| Apify | 云服务 | 多平台 | 无需维护基础设施 |
| Smartproxy | 代理+API | 多平台 | 内置反爬防护 |

## 六、数据存储方案

- **结构化数据**：CSV / JSON / SQLite
- **大规模采集**：PostgreSQL + 定时任务
- **流式数据**：Redis Queue → 消费者写入 DB
