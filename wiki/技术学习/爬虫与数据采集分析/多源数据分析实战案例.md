## 二、数据清洗与预处理

### 2.1 探索性数据分析（EDA）

```python
import pandas as pd

df = pd.read_csv("reviews.csv")
print(df.shape)           # 数据规模
print(df.isnull().sum())  # 缺失值检查
print(df.duplicated().sum())  # 重复行数

# 清洗
df.dropna(subset=['content'], inplace=True)
df.drop_duplicates(subset=['content'], inplace=True)
df['content'] = df['content'].str.strip()
```

### 2.2 NLP 预处理步骤

| 步骤 | 操作 | 工具 |
|------|------|------|
| 文本标准化 | 去除特殊字符、统一小写 | re, str.lower() |
| 分词 | 切分为单词/子词 | NLTK, spaCy |
| 去除停用词 | 过滤无意义词汇 | NLTK stopwords |
| 词干提取 | 还原词根（acting → act） | Porter Stemmer |
| 特征提取 | 文本转数值向量 | TF-IDF, CountVectorizer |

```python
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer
import re

stop_words = set(stopwords.words('english'))
stemmer = PorterStemmer()

def preprocess(text: str) -> str:
    text = re.sub(r'[^a-zA-Z\s]', '', text.lower())
    tokens = word_tokenize(text)
    tokens = [stemmer.stem(w) for w in tokens if w not in stop_words]
    return ' '.join(tokens)
```

## 三、情感分析方案对比

### 3.1 VADER（词典方法，快速）

```python
from nltk.sentiment import SentimentIntensityAnalyzer

sia = SentimentIntensityAnalyzer()
scores = sia.polarity_scores("This product is absolutely amazing!")
# {'neg': 0.0, 'neu': 0.254, 'pos': 0.746, 'compound': 0.6369}
```

- **优势**：不需要训练数据，速度快
- **劣势**：无法识别语境和讽刺

### 3.2 RoBERTa / BERT（深度学习，精准）

```python
from transformers import pipeline

# 预训练好的情感分析模型
classifier = pipeline(
    "sentiment-analysis",
    model="cardiffnlp/twitter-roberta-base-sentiment"
)

result = classifier("I can't believe how terrible this was...")
# [{'label': 'NEGATIVE', 'score': 0.9823}]
```

- **优势**：理解语境和讽刺，准确率显著高于词典方法
- **劣势**：需要 GPU/较多计算资源

### 3.3 方案选型建议

| 场景 | 推荐方案 |
|------|----------|
| 快速原型、数据量小 | VADER |
| 生产环境、需要高精度 | RoBERTa / FinBERT |
| 中文评论分析 | ERNIE / 中文 BERT |
| 实时流式处理 | VADER / DistilBERT（轻量） |

## 六、部署与自动化

```
数据管道架构：
定时触发（Cron/n8n）
    → 爬虫脚本采集新数据
    → 数据清洗 + NLP 推理
    → 结果写入数据库（PostgreSQL/BigQuery）
    → 自动生成报告（PDF/邮件）
    → 异常指标触发 Slack 告警
```

**技术栈建议**：
- **调度**：Airflow / n8n / GitHub Actions
- **存储**：PostgreSQL（结构化）+ S3（原始数据）
- **推理**：FastAPI + ONNX（模型服务化）
- **可视化**：Metabase / Grafana
